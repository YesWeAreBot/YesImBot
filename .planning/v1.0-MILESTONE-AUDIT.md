## Integration Check

### Score: 18/21 wiring points connected

**Connected (18):** All core service wiring verified — HorizonService→AgentCore, PromptService→ThinkActLoop, PluginService→buildAiSdkTools, WillingnessEngine→gateAndEnqueue, ModelService→providers, Schema.dynamic→registry.chatModels.

**Issues (3):**

1. **Non-stream path bypasses ModelService** — `loop.ts:92` calls `generateText()` directly instead of `modelService.call()`. PQueue concurrency control and fallback chain are skipped for the default (non-stream) path. Affects: AGENT-01, AGENT-03, MODEL-01.

2. **parseModelId unused / fallbackModel dead** — `loop.ts:51` imports `parseModelId` but never calls it. `config.fallbackModel` is declared in `AgentCoreConfig` but never read. If `config.model` is invalid, the loop crashes with no fallback. Affects: MODEL-04, MODEL-05.

3. **Provider ctx.get() pattern** — `provider-openai` and `provider-deepseek` declare `inject = ["yesimbot.model"]` but also use `ctx.get("yesimbot.model")` in `apply()`. The `inject` ensures load order, so `ctx.get()` is redundant but functional. Affects: PLATFORM-01.
