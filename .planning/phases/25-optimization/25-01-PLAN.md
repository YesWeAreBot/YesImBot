**In `core/src/services/agent/loop.ts`:**

Modify the wmLines builder loop (around line 120-135) to implement OPT-03 triggeredAt and OPT-04 send_message trimming.

1. **Derive channelKey** at the top of the `try` block (after `view` is built):
   ```typescript
   const channelKey = `${percept.scope.platform}:${percept.scope.channelId}`;
   ```

2. **Build triggeredAt labels (OPT-03):**
   In the wmLines loop, for each `agent.response` observation, find the last `message` observation that precedes it in `view.history`. Use `horizon.getShortId(channelKey, triggerMsg.messageId)` to get the short ID. Format:
   ```
   Round N (triggered by #M):
   ```
   If no preceding message found, omit the triggered-by suffix (just `Round N:`).

   Implementation approach:
   ```typescript
   const wmLines: string[] = [];
   for (let i = 0; i < (view.history ?? []).length; i++) {
     const obs = view.history![i];
     if (obs.type === "agent.response") {
       const d = obs.data;
       // Find last message before this agent response
       let triggerLabel = "";
       for (let j = i - 1; j >= 0; j--) {
         const prev = view.history![j];
         if (prev.type === "message") {
           const shortId = horizon.getShortId(channelKey, prev.messageId);
           if (shortId !== undefined) {
             triggerLabel = ` (triggered by #${shortId})`;
           }
           break;
         }
       }
       const lines = [`Round ${d.round}${triggerLabel}:`];
       // ... action lines
     }
   }
   ```

3. **send_message trimming (OPT-04):**
   Inside the action iteration within the wmLines builder, special-case `send_message`:
   ```typescript
   for (const a of d.actions) {
     const r = d.toolResults.find((t) => t.name === a.name);
     if (a.name === "send_message") {
       // OPT-04: omit content param, compact result
       const ok = r?.status === "ok" || r?.status === "fulfilled" || (r && !r.error);
       if (ok) {
         lines.push("  - send_message({}) -> sent, ok");
       } else {
         const errMsg = r?.error ?? "unknown";
         lines.push(`  - send_message({}) -> sent, failed: ${errMsg}`);
       }
     } else {
       // Existing logic for other tools — unchanged
       const status = r ? r.status + (r.error ? ": " + r.error : "") : "no result";
       const preview = r?.result != null ? String(r.result).slice(0, 200) : "";
       lines.push(
         `  - ${a.name}(${JSON.stringify(a.params ?? {})}) -> ${status}${preview ? ": " + preview : ""}`,
       );
     }
   }
   ```

   Per CONTEXT.md locked decision: "始终省略 send_message 的内容参数" — always omit content. Success = `sent, ok`. Failure = `sent, failed: reason`. Note: we don't include `#N` target message ID in this plan because the send_message tool result doesn't currently return the sent message's platform ID. Using the simpler `sent, ok` format is correct per the research open question. If the tool is later updated to return the ID, the format can be enhanced.

**Do NOT modify** any other part of loop.ts (the model call, trimming, parsing, etc.).
