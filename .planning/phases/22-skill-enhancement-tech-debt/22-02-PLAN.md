  <name>Task 2: Implement trait-bound lifecycle in resolve() (DEBT-02)</name>
  <files>
    core/src/services/skill/service.ts
  </files>
  <action>
Add a trait-bound branch to `resolve()` in service.ts, parallel to the existing sticky logic. The trait-bound lifecycle persists active state in channelState and re-evaluates conditions each turn, removing immediately when conditions are unmet.

**In the `if (activated)` block (after the existing sticky branch around line 105-111), add:**
```typescript
} else if (skill.lifecycle === "trait-bound") {
  state.set(skill.name, {
    lifecycle: "trait-bound",
    roundsSinceActive: 0,
    stickyTimeout: 0, // unused for trait-bound
  });
}
```

**In the `else` branch (after the existing sticky countdown around line 112-120), add a trait-bound removal branch:**
```typescript
} else if (skill.lifecycle === "trait-bound" && state.has(skill.name)) {
  // Trait signal gone → immediate removal, no grace period
  state.delete(skill.name);
  this.logger.info("trait-bound skill %s deactivated (trait signal lost)", skill.name);
}
```

**Key behavioral differences from sticky:**
- Sticky: counts down `roundsSinceActive` and keeps skill active until timeout
- Trait-bound: immediate removal when condition is unmet (no grace period, per user decision)
- Both share the same `channelState` Map, distinguished by the `lifecycle` field in `ActiveSkillState`

**Logging:** Add lifecycle type to the existing activation log for observability:
```typescript
this.logger.info("skill %s activated (lifecycle: %s)", skill.name, skill.lifecycle);
```
