# Plan 14-01: Provider Pattern Cleanup & PLATFORM-01

**Phase Goal**: Remove redundant `ctx.get()` from providers, use idiomatic inject pattern, close PLATFORM-01

## Context

Both `provider-openai` and `provider-deepseek` already declare `inject = ["yesimbot.model"]`, which guarantees `ctx["yesimbot.model"]` exists when `apply()` runs. Yet both still use `ctx.get("yesimbot.model")` with a manual null check — this is redundant and non-idiomatic.

Additionally, neither package.json declares `koishi.service.required`, which is the Koishi convention for declaring service dependencies in package metadata.

## Tasks

### Task 1: Fix provider-openai inject pattern
**File**: `providers/provider-openai/src/index.ts`
**Changes**:
- Replace `ctx.get("yesimbot.model") as IModelService` with `ctx["yesimbot.model"]`
- Remove the null check (`if (!modelService)` block)
- Remove unused `IModelService` import if no longer referenced

### Task 2: Fix provider-deepseek inject pattern
**File**: `providers/provider-deepseek/src/index.ts`
**Changes**:
- Same as Task 1 — replace `ctx.get()`, remove null check, clean unused import

### Task 3: Add koishi.service metadata to package.json files
**Files**:
- `providers/provider-openai/package.json`
- `providers/provider-deepseek/package.json`
**Changes**:
- Add `"koishi": { "service": { "required": ["yesimbot.model"] } }` to each

### Task 4: Verify build
- Run `yarn typecheck` for both provider packages
- Confirm no type errors

## Verification

- [ ] No `ctx.get(` calls remain in provider source files
- [ ] Both providers use `ctx["yesimbot.model"]` directly
- [ ] Both package.json files declare `koishi.service.required`
- [ ] `yarn typecheck` passes for both packages
