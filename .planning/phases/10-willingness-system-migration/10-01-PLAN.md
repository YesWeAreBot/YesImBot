**Complete rewrite** of `willingness.ts`. Remove the old `WillingnessCalculator` class entirely. Create `WillingnessEngine`.

**Per-channel state** (Map keyed by `platform:channelId`):
```typescript
interface ChannelState {
  willingness: number;
  lastMessageAt: number;
  botReplyTimestamps: number[];
}
```

**WillingnessEngine class:**

Constructor takes `WillingnessConfig`. Pre-compile keyword regexes from `config.gain.keywords` into a `RegExp[]` array cached on the instance.

Compute `baseFactor = Math.pow(0.5, 1 / config.decay.halfLife)` once in constructor.

**Public methods:**

1. `tick()` ‚Äî called every 1s by external timer. For each channel state: compute effective decay factor via `computeDecayFactor()`, multiply willingness by it. Remove channels where willingness rounds to 0 and botReplyTimestamps is empty.

2. `processMessage(channelKey: string, triggerType: TriggerType, content: string): { probability: number; shouldReply: boolean }` ‚Äî the main decision method:
   - Get or create channel state
   - Update `lastMessageAt = Date.now()`
   - Compute base gain (`config.gain.baseGain`)
   - If any keyword regex matches content: multiply gain by `config.gain.keywordMultiplier`
   - Apply sigmoid gain multiplier: `gain *= sigmoidGainMultiplier(state.willingness, config.maxWillingness, config.sigmoid.midpoint, config.sigmoid.steepness)`
   - Add gain to `state.willingness`, clamp to `[0, maxWillingness]`
   - Convert to probability: `probability = state.willingness / config.maxWillingness` (linear mapping)
   - Apply fatigue penalty: `probability *= computeFatiguePenalty(...)`
   - If triggerType is "mention" or "reply": `probability = applyMentionBoost(probability, config.mentionBoost)`
   - `shouldReply = Math.random() < probability`
   - Return `{ probability, shouldReply }`

3. `recordBotReply(channelKey: string)` ‚Äî push `Date.now()` to `state.botReplyTimestamps`

**Pure helper functions** (module-level, not methods):

- `computeDecayFactor(baseFactor, willingness, elasticThreshold, maxWillingness, silenceMs)` ‚Äî elastic decay + four-tier heat detection. Use exact formulas from RESEARCH.md code examples.
- `sigmoidGainMultiplier(current, max, midpoint, steepness)` ‚Äî inverted sigmoid scaled to [0, 2]. Use exact formula from RESEARCH.md.
- `computeFatiguePenalty(botReplyTimestamps, now, windowMs, threshold, penaltyBase)` ‚Äî prune old timestamps, compute exponential penalty. Use exact formula from RESEARCH.md.
- `applyMentionBoost(baseProbability, mentionBoost)` ‚Äî locked formula `P = base + (1 - base) * mentionBoost`.

Import `TriggerType` from `../horizon/types`.

**Anti-patterns to avoid:**
- Do NOT store Session objects in state (only primitives)
- Do NOT use `any` types
- Prune `botReplyTimestamps` on every access to prevent unbounded growth
