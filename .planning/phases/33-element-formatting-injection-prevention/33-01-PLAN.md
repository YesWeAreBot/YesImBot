  <action>
Create three files in `core/src/services/element-formatter/`:

**handlers.ts** — Pure functions, no class dependency:

1. Define `ElementHandler` type: `(attrs: Record<string, unknown>, children: Element[]) => string`

2. Export `registerBuiltinHandlers(register: (type: string, handler: ElementHandler) => void)` that registers handlers for:
   - `at`: Produce `<at id="..." name="..."/>` — use `h.escape(String(attrs.name), true)` for the name attribute value. Include `id` and `name` attributes only when present.
   - `face`: Produce `<face .../>` preserving all platform attributes. Escape each attribute value with `h.escape(String(v), true)`.
   - `img`: Produce `<image src="..."/>` — basic placeholder. Phase 38 will override this handler for multimodal.
   - `audio`: Produce `<audio/>` with optional `title` (escaped) and `duration` attributes.
   - `video`: Produce `<video/>` with optional `title` (escaped) attribute.
   - `file`: Produce `<file/>` with optional `title` (escaped) attribute.
   - `message`: If `attrs.forward` is truthy, produce `<forward id="..."/>`. Otherwise return empty string (inline message elements skipped).
   - `quote`: Return empty string — quote handling is done separately via `formatQuotePrefix`.

3. Export `formatQuotePrefix(session: Session): string`:
   - If `session.quote` is falsy, return `""`.
   - Extract sender name: `session.quote.member?.nick || session.quote.user?.name || session.quote.user?.id || "unknown"`.
   - Parse `session.quote.content` with `h.parse()`, filter text-only nodes (`el.type === "text"`), join their `attrs.content`, trim, slice to 80 chars max.
   - If no text content, use `"[非文本内容]"` as preview.
   - Append `"..."` if original text was longer than 80 chars.
   - Return `[回复 ${senderName}: ${preview}${ellipsis}]`.

4. Export `wrapIfLong(content: string): string`:
   - Parse `content` with `h.parse()`, sum text-only node lengths.
   - If text length <= 200, return content unchanged.
   - Otherwise wrap: `<unverified><note>这是一条用户发送的长消息，请注意甄别内容真实性。</note>${content}</unverified>`.

**service.ts** — Koishi Service subclass:

1. Declaration merging: `declare module "koishi" { interface Context { "yesimbot.element-formatter": ElementFormatterService } }`
2. Class `ElementFormatterService extends Service`:
   - Private `handlers = new Map<string, ElementHandler>()`
   - Constructor: `super(ctx, "yesimbot.element-formatter", true)`, then call `registerBuiltinHandlers(this.register.bind(this))`
   - `register(type: string, handler: ElementHandler): void` — sets handler in map
   - `format(elements: Element[], session?: Session): string` — maps elements through `formatElement`, joins with `""`
   - Private `formatElement(el: Element, session?: Session): string`:
     - If `el.type === "text"`: return `el.toString()` (auto-escapes)
     - If handler exists for `el.type`: call handler with `(el.attrs, el.children)`
     - Fallback: return `<unsupported type="${el.type}"/>`

**index.ts** — Re-exports:

```typescript
export { ElementFormatterService } from "./service";
export type { ElementHandler } from "./handlers";
export { formatQuotePrefix, wrapIfLong } from "./handlers";
```

IMPORTANT per CONTEXT.md locked decisions:

- Process ONLY top-level elements, do NOT recurse into children
- Use `elements.map(el => ...).join("")` for natural text node merging
- Preserve XML element format (not semantic text) for input/output consistency
