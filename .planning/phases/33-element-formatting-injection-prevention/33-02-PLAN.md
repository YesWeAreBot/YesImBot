2. Rewrite `recordUserMessage(session)` to use the formatting pipeline:

   ```typescript
   private async recordUserMessage(session: Session): Promise<void> {
     this.logger.info(
       `user message | ${session.author?.name} | ${session.cid} | ${session.content}`,
     );
     if (session.guildId) {
       await this.updateMemberInfo(session, `guild:${session.guildId}`);
     } else if (session.isDirect) {
       await this.updateMemberInfo(session, `direct:${session.platform}`);
     }

     // Format elements through ElementFormatterService
     const formatter = this.ctx["yesimbot.element-formatter"];
     const elements = session.elements?.length ? session.elements : h.parse(session.content ?? "");
     let formattedContent = formatter.format(elements, session);

     // Prepend quote prefix if replying
     const quotePrefix = formatQuotePrefix(session);
     if (quotePrefix) {
       formattedContent = quotePrefix + " " + formattedContent;
     }

     // Wrap long messages with <unverified> tag
     formattedContent = wrapIfLong(formattedContent);

     await this.events.recordMessage({
       platform: session.platform,
       channelId: session.channelId ?? "",
       stage: TimelineStage.New,
       timestamp: new Date(session.timestamp),
       data: {
         messageId: session.messageId ?? "",
         senderId: session.author?.id ?? session.userId ?? "",
         senderName: session.author?.nick || session.author?.name || session.userId || "",
         content: formattedContent,
         replyTo: session.quote?.id,
       },
     });
   }
   ```

   Key changes from current code:
   - `session.content ?? ""` replaced by `formattedContent` (formatter output)
   - Added `replyTo: session.quote?.id` to store the quoted message ID
   - Added quote prefix and unverified wrapper in the pipeline
   - Fallback: if `session.elements` is empty/undefined, parse `session.content` with `h.parse()`

3. Also update the `horizon/message` event emission in the middleware to use `formattedContent`:
   In the middleware callback, the `content` field in the `payload` should also use the formatted content. Extract the formatting logic so it runs once and is used in both `recordUserMessage` and the event emission.

   Approach: Move the formatting into `recordUserMessage` and have it return the formatted content string. Then use that return value in the middleware for the event payload.

   Change `recordUserMessage` signature to `private async recordUserMessage(session: Session): Promise<string>` — returns the formatted content.

   In the middleware, capture the return:

   ```typescript
   const formattedContent = await this.recordUserMessage(session);
   await next();
   this.ctx.emit("horizon/message", {
     ...
     payload: {
       messageId: session.messageId ?? "",
       senderId: session.author?.id ?? session.userId ?? "",
       senderName: session.author?.nick || session.author?.name || session.userId || "",
       content: formattedContent,
     },
     ...
   });
   ```

CRITICAL anti-patterns to avoid (from RESEARCH.md):

- Do NOT call `h.parse(session.content)` and then `.toString()` on the result — this re-parses user text as XML elements, which is the injection vector
- Do NOT call `h.escape()` on the entire formatted output — this would double-escape element tags
- Do NOT re-parse `obs.content` in formatObservation() — it's already safe
- Use `session.elements` as the primary source (already parsed by adapter, text nodes are safe)
