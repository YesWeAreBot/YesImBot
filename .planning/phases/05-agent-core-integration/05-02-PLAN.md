  <action>
Create `plugins/core/src/services/agent/loop.ts`:
- Export `ThinkActLoop` class. Constructor takes: `ctx: Context` (Koishi context for service access).

- `async run(percept: Percept, config: AgentCoreConfig)`: The main entry point.
  1. Get services from ctx: `ctx["yesimbot.horizon"]` (HorizonService), `ctx["yesimbot.plugin"]` (PluginService), `ctx["yesimbot.prompt"]` (PromptService), `ctx["model-service"]` (ModelService).

  2. **Percept type guard (IMPORTANT):** Before calling buildView, check that percept is a UserMessagePercept. Check `percept.type === "user_message"` (or whatever the discriminant field is â€” read the Percept/UserMessagePercept types from horizon). If the check fails, log a warning `"Ignoring non-UserMessage percept: {percept.type}"` and return early. Only after the guard passes, cast: `const userPercept = percept as UserMessagePercept`.

  3. Build HorizonView: `const view = await horizon.buildView(userPercept)`.
  4. Build system prompt: `const systemPrompt = await prompt.render("system", { view })`. (PromptService.render already exists.)
  5. Build context text: `const contextText = horizon.formatHorizonText(view)`.
  6. Build FunctionContext: `{ session: percept.runtime?.session, view, percept }`.
  7. Build ai-sdk tools: call `buildAiSdkTools(pluginService, fnCtx, config.maxToolResultLength ?? 4000)` from tools.ts, spread with `finish: finishTool`.
  8. Build messages array: `[{ role: "user" as const, content: contextText }]`.
  9. Build stopWhen: `buildStopCondition(config.maxRounds ?? 3)` from tools.ts.

  10. **Route LLM calls through ModelService (CRITICAL â€” do NOT call generateText/streamText directly):**

      ModelService.call() accepts `(providerName, modelId, params: CallSettings & Prompt)`. Internally it does `generateText({ model, ...merged })` where `merged = { ...defaults, ...params }`. The ai-sdk `generateText` function destructures known fields and spreads the rest via `...settings`. This means extra properties like `tools`, `toolChoice`, `stopWhen`, `onStepFinish` pass through at runtime.

      Build the params object with all fields needed:

      ```ts
      const agentParams = {
        system: systemPrompt,
        messages,
        tools: allTools,
        toolChoice: "required" as const,
        stopWhen,
      } as CallParams;
      ```

      The `as CallParams` cast is needed because CallParams (= CallSettings & Prompt) doesn't explicitly type `tools`/`toolChoice`/`stopWhen`, but these properties pass through to generateText/streamText at runtime via object spread.

      If `config.streamMode`:
      - Call `const result = await modelService.streamCall(config.provider, config.model, agentParams)`.
      - `await result.text` to consume the stream.
      - Extract steps from `await result.steps`.
        Else (default):
      - Call `const result = await modelService.call(config.provider, config.model, agentParams)`.
      - Steps available as `result.steps`.

  11. Wrap the modelService call in a `Promise.race` with `setTimeout` for `config.globalTimeout ?? 120000` â€” reject with "Global loop timeout" on expiry.

  12. After loop completes, build AgentSummary string: concatenate tool names called across all steps + any send_message content. Simple format: `"Tools: [tool1, tool2]. Sent: [message content or 'nothing']"`.
  13. Record summary: `await horizon.events.recordAgentSummary({ scope: percept.scope, timestamp: new Date(), summary })`.
  14. Log completion with step count.

- Use a logger created from `ctx.logger("agent")`.

Enhance `plugins/core/src/services/plugin/builtin/send-message.ts`:

- Add `target` parameter to schema: `Schema.string().description("Target channel in platform:channelId format. Defaults to current channel.")`. Make it optional.
- In `sendMessage` handler:
  - Extract `content` and `target` from params.
  - Split content on `<sep/>`: `const parts = String(content).split("<sep/>").map(s => s.trim()).filter(Boolean)`.
  - If `target` is provided and differs from current channel:
    - Parse `target` as `platform:channelId` (split on first `:`)
    - Find bot: `this.ctx.bots.find(b => b.platform === platform)`
    - Send each part via `bot.sendMessage(channelId, part)`
  - Else (default â€” current channel):
    - Send each part via `ctx.session.send(part)`
  - Return Success().
