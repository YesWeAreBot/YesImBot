Create `plugins/core/src/services/agent/loop.ts`:
- Export `ThinkActLoop` class. Constructor takes: `ctx: Context` (Koishi context for service access).

- `async run(percept: Percept, config: AgentCoreConfig)`: The main entry point.
  1. Get services from ctx: `ctx["yesimbot.horizon"]` (HorizonService), `ctx["yesimbot.plugin"]` (PluginService), `ctx["yesimbot.prompt"]` (PromptService), `ctx["model-service"]` (ModelService).
  2. Build HorizonView: `const view = await horizon.buildView(percept as UserMessagePercept)`.
  3. Build system prompt: `const systemPrompt = await prompt.render("system", { view })`. (PromptService.render already exists.)
  4. Build context text: `const contextText = horizon.formatHorizonText(view)`.
  5. Build FunctionContext: `{ session: percept.runtime?.session, view, percept }`.
  6. Build ai-sdk tools: call `buildAiSdkTools(pluginService, fnCtx, config.maxToolResultLength ?? 4000)` from tools.ts, spread with `finish: finishTool`.
  7. Get the ai-sdk model object: `const { model } = modelService.getModel(config.provider ?? modelService.config.defaultProvider, config.model ?? modelService.config.defaultModel)`. (ModelService.getModel returns `{ model, defaultParams }`.)
  8. Build messages array: `[{ role: "user" as const, content: contextText }]`.
  9. Build stopWhen: `buildStopCondition(config.maxRounds ?? 3)` from tools.ts.

  10. If `config.streamMode`:
      - Call `streamText({ model, system: systemPrompt, messages, tools, toolChoice: "required", stopWhen })`.
      - `await result.text` to consume the stream (tool execution happens inside execute callbacks).
      - After stream consumed, extract steps from `await result.steps`.
  11. Else (default):
      - Call `generateText({ model, system: systemPrompt, messages, tools, toolChoice: "required", stopWhen })`.
      - Steps available as `result.steps`.

  12. Wrap the generateText/streamText call in a `Promise.race` with `setTimeout` for `config.globalTimeout ?? 120000` — reject with "Global loop timeout" on expiry.

  13. After loop completes, build AgentSummary string: concatenate tool names called across all steps + any send_message content. Simple format: `"Tools: [tool1, tool2]. Sent: [message content or 'nothing']"`.
  14. Record summary: `await horizon.events.recordAgentSummary({ scope: percept.scope, timestamp: new Date(), summary })`.
  15. Log completion with step count.

- Use a logger created from `ctx.logger("agent")`.

Enhance `plugins/core/src/services/plugin/builtin/send-message.ts`:

- Add `target` parameter to schema: `Schema.string().description("Target channel in platform:channelId format. Defaults to current channel.")`. Make it optional.
- In `sendMessage` handler:
  - Extract `content` and `target` from params.
  - Split content on `<sep/>`: `const parts = String(content).split("<sep/>").map(s => s.trim()).filter(Boolean)`.
  - If `target` is provided and differs from current channel:
    - Parse `target` as `platform:channelId` (split on first `:`)
    - Find bot: `this.ctx.bots.find(b => b.platform === platform)`
    - Send each part via `bot.sendMessage(channelId, part)`
  - Else (default — current channel):
    - Send each part via `ctx.session.send(part)`
  - Return Success().
