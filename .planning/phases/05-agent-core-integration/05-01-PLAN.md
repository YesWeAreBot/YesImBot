Create `plugins/core/src/services/agent/tools.ts`:

- Import `tool`, `jsonSchema`, `hasToolCall`, `stepCountIs` from "ai".
- Import `schemaToJSONSchema` from the plugin schema module.
- Import `PluginService` and `FunctionContext` from the plugin service.

- Export `finishTool`: an ai-sdk `tool()` with description "Signal that you have completed your response. Call this when done.", parameters as JSON schema with optional `summary: string`, execute returns `{ done: true, summary }`.

- Export `buildAiSdkTools(pluginService: PluginService, fnCtx: FunctionContext, maxResultLength: number)`: iterates `pluginService.getTools()` (which returns OpenAI-format array), for each entry creates an ai-sdk `tool()` with:
  - `description` from the function definition
  - `parameters: jsonSchema(entry.function.parameters)` (already JSON schema from getTools)
  - `execute`: calls `pluginService.invoke(name, params, fnCtx)`, returns `result.content` on success or `{ error: result.error }` on failure. If stringified result exceeds `maxResultLength`, truncate and append "[truncated]".

- Export `buildStopCondition(maxRounds: number)`: returns `[hasToolCall("finish"), stepCountIs(maxRounds)]`.

Note: `getTools()` already returns JSON Schema (via `schemaToJSONSchema`), so `tools.ts` passes it directly to `jsonSchema()` wrapper — no double conversion needed.

Create `plugins/core/src/services/agent/service.ts`:
- Declare module "koishi" extending Context with `"yesimbot.agent": AgentCore`.
- `AgentCore extends Service<AgentCoreConfig>` with `static inject = ["yesimbot.horizon", "yesimbot.plugin", "yesimbot.prompt", "model-service"]`.
- Constructor: `super(ctx, "yesimbot.agent", false)` (async init — waits for `start()`).

- Private fields:
  - `queues: Map<string, Promise<void>>` — per-channel promise chain for session isolation.
  - `pending: Map<string, Percept>` — latest queued Percept per channel (for merge/replace).

- `start()`: Listen on `horizon/percept` event via `this.ctx.on("horizon/percept", (percept) => this.handlePercept(percept))`. Log "AgentCore started".

- `handlePercept(percept: Percept)`: Compute `channelKey` from `percept.scope` as `platform:channelId`. Check if a loop is currently running for this channel (queues has an entry that hasn't resolved). If running, replace `pending.get(channelKey)` with the new percept (latest wins — per decision). If not running, call `enqueue(channelKey, percept)`.

- `enqueue(channelKey: string, percept: Percept)`: Chain onto `queues.get(channelKey) ?? Promise.resolve()`. The chained function calls `this.runLoop(percept)`, then checks `pending.get(channelKey)` — if a pending percept exists, delete it from pending and recursively enqueue it. Catch errors and log them. Update `queues` map. When the chain fully resolves, delete the channel key from `queues`.

- `runLoop(percept: Percept)`: Placeholder method that logs "runLoop called for {channelKey}" — actual implementation in Plan 02. Returns `Promise<void>`.

Create `plugins/core/src/services/agent/index.ts`:

- Barrel exports: `export { AgentCore } from "./service"`, `export type { AgentCoreConfig, AgentIdentity } from "./config"`.
