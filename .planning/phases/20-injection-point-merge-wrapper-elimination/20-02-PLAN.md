  <action>
**Constructor changes:**
- Remove the seeding logic that checks for `system.mustache` existence (the `if (this.resourcesDir !== builtinResourcesDir && !existsSync(...))` block). Replace with a check for `core-memory.mustache` instead — this is the only retained template that MemoryService needs seeded.
- Remove `this.registerTemplate("system", ...)` — no system template needed.
- Remove the `partialMap` block that registers 8 partials. Replace with registering ONLY the 2 retained partials: `"memory-block"` and `"horizon-view"` (these are still used by MemoryService and HorizonService respectively).
- Remove ALL 4 default injection registrations (`__default_identity`, `__default_control_flow`, `__default_basic_functions`, `__default_style`). Phase 21 will provide content via SOUL.md/AGENTS.md. Empty output is acceptable per user decision.

**render() rewrite:**
- Remove the `templateName` parameter dependency — render no longer needs a template. Keep the signature for backward compat but ignore `templateName`.
- Remove `buildScope()` call that parses template variables. Instead, build scope directly from `initialScope` and evaluate ALL registered snippets unconditionally (no template-based filtering).
- After collecting fragments per injection point, generate XML inline:
  ```
  const content = fragments.join("\n\n");
  sections.push({
    name: point,
    content: `<${point}>\n${content}\n</${point}>`,
    cacheable: true,
  });
  ```
- Always emit the tag even if content is empty (user decision: structural consistency).
- Remove the second loop that rendered via partials — the single loop now does everything.
- Remove `scope[point + "_content"]` and `scope["has_" + point]` assignments — no templates consume these.

**Cleanup:**
- Remove `getRequiredVariables()` private method (no longer needed without template parsing).
- Remove `isSnippetRequired()` private method (all snippets evaluated unconditionally).
- The `buildScope()` method can be simplified to just evaluate all snippets without the `required` filter. Or inline it into render() if cleaner.
- Keep `loadTemplate()`, `loadPartial()`, `registerPartial()` — still used by MemoryService.
- Keep `templates` Map and `registerTemplate()` — may be used by other services.
- Keep `renderer` (MustacheRenderer) — still used for variable interpolation within snippet content.
