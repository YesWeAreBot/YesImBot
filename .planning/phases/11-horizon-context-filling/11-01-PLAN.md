  <action>
**types.ts changes:**
- No structural changes needed — existing EntityRecord.attributes already supports arbitrary fields (avatar, lastActive stored there). Existing Environment type already has name/metadata fields.

**config.ts changes:**
- Add to HorizonServiceConfig: `botName?: string` (config-priority bot display name), `entityCacheTtl?: number` (ms, default 3600000 = 1h), `maxActiveEntities?: number` (default 15).

**listener.ts changes — throttled entity updates:**
- Add a `private lastEntityUpdate = new Map<string, number>()` for throttling. In `updateMemberInfo()`, check if `Date.now() - lastEntityUpdate.get(id) < 60000` (1 min window) — if so, skip the DB write. Otherwise update the timestamp in the map.
- Extend the data written in `updateMemberInfo()` to include `attributes: { roles, platform, avatar: session.author.avatar, lastActive: new Date() }`.
- Add DM entity recording: in `recordUserMessage()`, if `session.isDirect` (no guildId), also call `updateMemberInfo()` using a synthetic parentId like `direct:${session.platform}` instead of `guild:${guildId}`. The entity id format for DM: `${session.platform}:${session.author.id}@direct:${session.platform}`.
- Use `ctx.database.upsert` instead of the get-then-set/create pattern in `updateMemberInfo()` for simplicity. Koishi's database API supports upsert on the entity table.

**service.ts changes — Environment lazy-load:**
- Rename `getEnvironment()` to a private `getOrCreateEnvironment()` method. Logic:
  1. Build id as `${scope.platform}:${scope.channelId}`
  2. Query DB for entity with that id and type "channel"
  3. If found AND `updatedAt` is within `config.entityCacheTtl` (default 1h), return as Environment
  4. If not found or stale: attempt to get real name from the percept's `runtime.session`:
     - `channelName = session.channelName || session.guildName || null`
     - If still null and session.bot exists, try `await session.bot.getChannel(scope.channelId, scope.guildId).catch(() => null)` then use result's `name` field
     - Fallback name: `${scope.platform}:${scope.channelId}`
  5. Upsert entity record: `{ id, type: "channel", name: channelName, attributes: { platform: scope.platform, isDirect: scope.isDirect, guildId: scope.guildId }, updatedAt: new Date() }`
  6. Return Environment: `{ type: scope.isDirect ? "private" : "group", id, name: channelName, metadata: { platform: scope.platform } }`
- In `buildView()`: call `getOrCreateEnvironment(percept.scope, percept)` passing the percept so it can access `runtime.session`.
- In `buildView()` for self info: use `this.config.botName || session?.bot?.user?.name || session?.bot?.selfId || ""` for the name field.
- In `getEntities()`: add `.orderBy("updatedAt", "desc").limit(config.maxActiveEntities ?? 15)` to the query. Also handle DM case: if `!scope.guildId && scope.isDirect`, query with parentId `direct:${scope.platform}`.
  </action>
