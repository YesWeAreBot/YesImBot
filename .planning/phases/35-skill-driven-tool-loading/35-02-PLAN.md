**1. `types.ts`** ŌĆö SearchBackend interface and types:

```typescript
import type { Context, Schema } from "koishi";

export interface SearchResult {
  title: string;
  url: string;
  snippet: string;
}

export interface SearchOptions {
  limit?: number;
  [key: string]: unknown;
}

export interface SearchBackend {
  readonly name: string;
  search(query: string, options: SearchOptions): Promise<SearchResult[]>;
  getParameterSchema(): Record<string, Schema>;
}

export interface SearchPluginConfig {
  provider?: string;
  endpoint?: string;
  apiKey?: string;
  defaultLimit?: number;
}
```

**2. `backends/tavily.ts`** ŌĆö Tavily implementation:

- Class `TavilyBackend` implements `SearchBackend`
- `name` = `"tavily"`
- Constructor takes `ctx: Context` and `config: SearchPluginConfig`
- `search()` method:
  - Uses `ctx.http.post()` to call `config.endpoint` (default: `"https://api.tavily.com/search"`)
  - POST body: `{ api_key: config.apiKey, query, max_results: options.limit ?? config.defaultLimit ?? 5 }`
  - Maps response `results` array to `SearchResult[]`: `{ title: r.title ?? "", url: r.url ?? "", snippet: r.content ?? "" }`
  - Wraps in try/catch ŌĆö on ANY error, return empty array `[]` (silent degradation per user decision)
  - If `config.apiKey` is falsy, return empty array immediately (no HTTP call)
- `getParameterSchema()` returns `{ limit: Schema.number().default(5).description("Maximum number of results to return") }`

**3. `index.ts`** ŌĆö SearchPlugin:

- Import `Plugin` from `../../base-plugin`, `Metadata` from `../../decorators`, `withInnerThoughts` from `../../decorators`, `Success` from `../../utils`, `FunctionType` from `../../types`
- Import `SearchBackend`, `SearchPluginConfig` from `./types`
- Import `TavilyBackend` from `./backends/tavily`
- `@Metadata({ name: "search", description: "Web search tool", builtin: true })`
- Class `SearchPlugin extends Plugin`
- Constructor takes `ctx: Context` and `config: SearchPluginConfig`
  - Calls `super()`
  - Creates backend: if no config or no apiKey, still create TavilyBackend (it handles missing apiKey gracefully)
  - Currently only Tavily is supported, so just `const backend: SearchBackend = new TavilyBackend(ctx, config)`
  - Calls `this.registerTool()` with a `FunctionDefinition`:
    - `name: "search"`
    - `description: "Search the web for information. Returns a list of relevant results with titles, URLs, and snippets."`
    - `type: FunctionType.Tool`
    - `hidden: true`
    - `parameters: withInnerThoughts({ query: Schema.string().required().description("Search query"), ...backend.getParameterSchema() })`
    - `handler`: async function that:
      1. Extracts `query` from params as string
      2. Extracts `limit` from params as number | undefined
      3. Calls `backend.search(query, { limit })`
      4. If results empty, returns `Success("No results found.")`
      5. Otherwise formats as numbered list: `${i+1}. ${r.title}\n   ${r.url}\n   ${r.snippet}` joined by `\n\n`
      6. Returns `Success(formatted)`

Key points:

- Use `registerTool()` (not `@Tool` decorator) since the tool is dynamically constructed from backend schema
- The `hidden: true` flag ensures the tool is invisible without an active Skill
- No error handling needed in handler ŌĆö backend.search() already catches all errors and returns `[]`
