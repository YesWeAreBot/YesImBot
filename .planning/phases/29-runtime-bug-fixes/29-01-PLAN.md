<objective>
Replace the single-slot pending message Map with an array queue so burst messages accumulate instead of overwriting each other. After the in-flight loop completes, drain all backlogged payloads into one merged request.

Purpose: Eliminate message loss during response generation — users who send multiple messages while the bot is thinking will get a single combined follow-up response instead of losing all but the last message.
Output: Updated `service.ts` with array-based pending queue and mergeBacklog helper.
</objective>

   ```typescript
   private mergeBacklog(backlog: LoopPayload[]): LoopPayload {
     const first = backlog[0];
     if (backlog.length === 1) return first;

     const combinedContent = backlog
       .map((p) => p.percept.metadata?.content as string ?? "")
       .filter(Boolean)
       .join("\n");

     return {
       percept: {
         ...first.percept,
         metadata: {
           ...first.percept.metadata,
           content: combinedContent,
           isBacklogDrain: true,
           backlogCount: backlog.length,
         },
       },
       toolCtx: first.toolCtx,
     };
   }
   ```
   Key: uses spread for immutability (never mutate first.percept). Uses first message's timestamp per CONTEXT.md decision.
