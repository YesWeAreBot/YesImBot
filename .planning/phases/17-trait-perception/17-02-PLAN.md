Create `core/src/services/trait/detectors/scene.ts` — SceneTrait detector:

- Implements TraitDetector interface, `name = 'scene'`
- `start(ctx, analyzer)`: store analyzer ref, subscribe to `ctx.on('horizon/message', ...)` to track:
  - Last bot response timestamp/message-count-since per channel (for ignored detection)
  - Whether bot was mentioned in recent messages (for mentioned detection)
  - Create logger once: `const logger = ctx.logger('trait:scene')`
- Internal state type `SceneState`: `{ lastBotResponseAt?: number, messagesSinceBotResponse: number, lastMentionedAt?: number, messagesSinceMention: number }`
- Event handler: on each horizon/message, increment messagesSinceBotResponse and messagesSinceMention counters. If message mentions bot (check via view not available here — track from event payload content matching bot name), update lastMentionedAt and reset messagesSinceMention.
  - Bot name: get from `analyzer` or pass via config. Simplest: store bot name from first detect() call's `view.self.name` (lazy init).
- `detect(scope, view)`:
  - Scene group (mutually exclusive): `scope.isDirect ? 'private-chat' : 'group-chat'` → signal with dimension='scene', confidence=1.0
  - Attention group (mutually exclusive): check mentioned vs ignored
    - **mentioned**: check if any recent history message contains `@` + bot name OR bot name substring (case-insensitive exact match on `view.self.name`). Also check the percept trigger — if view has recent messages mentioning bot. Simple approach: scan last few messages in `view.history` for bot name.
    - **ignored**: two conditions (per user decision):
      1. Bot spoke (has agent.response in history) but messagesSinceBotResponse >= 5 (configurable)
      2. messagesSinceMention >= 10 (configurable)
      Either condition → ignored signal with confidence=0.8
    - If neither mentioned nor ignored, output no attention signal
  - Utility: `channelKey(scope)` → `${scope.platform}:${scope.channelId}` (reuse existing pattern)

Create `core/src/services/trait/detectors/heat.ts` — HeatTrait detector:

- Implements TraitDetector, `name = 'heat'`
- `start(ctx, analyzer)`: store analyzer ref, subscribe to `ctx.on('horizon/message', ...)` to push timestamps and prune old ones
  - Create logger once: `const logger = ctx.logger('trait:heat')`
- Internal state type `HeatState`: `{ timestamps: number[] }`
- Config constants (Claude's discretion per CONTEXT.md):
  - `windowMs = 5 * 60 * 1000` (5 minutes)
  - `highThreshold = 8` (msgs/min)
  - `mediumThreshold = 2` (msgs/min)
- Event handler: push `Date.now()` to timestamps array, prune entries older than windowMs
- `detect(scope, _view)`:
  - Read state, if none → return `[{dimension:'heat', value:'low', confidence:1.0}, {dimension:'heat-trend', value:'stable', confidence:1.0}]`
  - Calculate rate: `recentCount / (windowMs / 60000)` msgs per minute
  - Level: rate >= highThreshold → 'high', >= mediumThreshold → 'medium', else 'low'
  - Trend: split window at midpoint, compare recentHalf vs olderHalf counts. recentHalf > olderHalf * 1.3 → 'heating', recentHalf < olderHalf * 0.7 → 'cooling', else 'stable'
  - Return two signals: dimension='heat' + dimension='heat-trend'

IMPORTANT:
- Logger created once in start(), NOT chained on ctx.logger()
- No explicit `any` — use proper types
- channelKey helper: define locally or import from a shared utility
