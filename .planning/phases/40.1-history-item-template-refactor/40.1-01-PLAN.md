Purpose: Decouple rendering format from TypeScript code — template changes don't require code changes. Simplify XML tags (bot-action -> action, bot-error -> error, inline sender, DD:HH:MM time).
Output: Updated service.ts, history-item.mustache, horizon-view.mustache, updated tests.

1. In `service.ts`, define `HistoryItemData` interface above the class (or export from types.ts if preferred — but since it's only used by service.ts, keep it local):

```typescript
interface HistoryItemData {
  is_message: boolean;
  is_action: boolean;
  is_error: boolean;
  // message fields
  id?: number;
  time?: string; // "DD:HH:MM"
  senderLine?: string; // "SenderName(senderId)"
  replyLine?: string; // "[回复: N]" or undefined
  content?: string;
  // action fields
  actionContent?: string;
  // error fields
  errorContent?: string;
}
```

2. Change `formatObservation` return type from `string` to `HistoryItemData | null`. Rewrite each branch:

**message branch:**

- Remove the no-channelKey fallback path entirely — it is dead code. `formatObservation` is ONLY called from `formatHorizonText` which always passes channelKey. Delete the `if (channelKey)` conditional and the else branch. The method now requires channelKey (keep the parameter optional for signature compat, but the no-channelKey path simply returns `null`).
- Time format: `DD:HH:MM` using `d.getDate()`, `d.getHours()`, `d.getMinutes()` all zero-padded to 2 digits
- `senderLine`: `${senderName}(${senderId})` — same bot/badge logic as current
- `replyLine`: if `obs.replyTo` and shortId found, `[回复: ${replyShortId}]`, else undefined
- Return `{ is_message: true, is_action: false, is_error: false, id: shortId, time, senderLine, replyLine, content: obs.content }`

**agent.action branch:**

- Remove `round` and `trigger` attributes entirely
- `actionContent`: same `lines.join("; ")` logic as current
- Return `{ is_message: false, is_action: true, is_error: false, actionContent }`

**agent.response branch (error):**

- Rename from `<bot-error>` to `<error>` — this is now handled by the `is_error` branch in the template
- `errorContent`: `esc(obs.data.error)` — keep the esc() helper
- Return `{ is_message: false, is_action: false, is_error: true, errorContent }`

**agent.response branch (no error):**

- Return `null` (was returning empty string "")

**Default:**

- Return `null`

3. In `formatHorizonText`, update the null check: use `if (!item)` to skip null returns (not `if (!formatted)` on empty strings).

4. Completely replace `history-item.mustache` content. The current file contains incompatible old format (`<message>` tag, `{{times}}`, `{{senderName}}`, `{{senderId}}`). Overwrite with:

```mustache
{{/is_message}}{{#is_action}}<action>{{actionContent}}</action>
{{/is_action}}{{#is_error}}<error>{{errorContent}}</error>
{{/is_error}}
```

Note: Each type block ends with a newline. The template should produce one line per observation when rendered.
