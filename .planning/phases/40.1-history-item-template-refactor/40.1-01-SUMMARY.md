# Phase 40.1 Plan 01: History Item Template Refactor Summary

**Migrated history item rendering from hardcoded XML strings to Mustache partial template, simplified XML tags (action/error), and implemented inline sender format with DD:HH:MM timestamps**

## Performance

- **Duration:** 3 min
- **Started:** 2026-02-28T13:32:41Z
- **Completed:** 2026-02-28T13:35:41Z
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments

- **Template-based rendering**: history-item.mustache partial defines all observation type rendering formats
- **Structured data objects**: formatObservation returns HistoryItemData instead of XML strings
- **Simplified XML tags**: <bot-action> -> <action>, <bot-error> -> <error>, removed round/trigger attributes
- **Inline sender format**: <msg id="N" time="DD:HH:MM">SenderName(senderId) content</msg>
- **Reply inline format**: [回复: N] rendered inline before message content
- **Time format**: DD:HH:MM (day-of-month:hour:minute) with zero-padding
- **Trigger section updated**: horizon-view.mustache uses {{>history-item}} partial instead of {{{.}}}

## Files Created/Modified

- `core/resources/templates/partials/history-item.mustache` - Mustache partial for all observation types (message, action, error)
- `core/src/services/horizon/service.ts` - Added HistoryItemData interface, changed formatObservation return type, updated formatHorizonText to use partials
- `core/resources/templates/partials/horizon-view.mustache` - Updated trigger section to use {{>history-item}} partial
- `core/src/services/horizon/__tests__/format-horizon-text.test.ts` - Added test suite for history-item partial rendering

## Decisions Made

- **HistoryItemData interface location**: Defined outside HorizonService class because TypeScript doesn't allow interfaces inside classes
- **Triple mustache for actionContent**: Used {{{actionContent}}} to prevent HTML escaping of arrow characters (-> renders as -&gt; with double mustache)
- **No-channelKey path removal**: Deleted legacy fallback path in formatObservation (dead code - only called from formatHorizonText which always passes channelKey)
- **Null returns**: formatObservation returns null for successful agent.response observations (actions rendered separately via AgentActionObservation)

## Deviations from Plan

None - plan executed exactly as written.

## Issues Encountered

- **Mustache HTML escaping**: Initial test failed because actionContent used double mustache {{actionContent}}, which escaped -> to -&gt;. Fixed by changing to triple mustache {{{actionContent}}}.
- **Test type errors**: TypeScript complained about assigning HistoryItemData objects to history/trigger arrays (typed as never[]). Fixed by casting to unknown[].
- **Template variable scope**: New test suite failed because template variable wasn't defined in that scope. Fixed by loading horizonViewTpl separately.
