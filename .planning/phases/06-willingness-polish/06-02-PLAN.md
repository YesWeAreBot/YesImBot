In `send-message.ts`:
- Import `sleep` from "koishi".
- Add inter-part delay: when iterating `effectiveParts` and there are multiple parts, insert `await sleep(1000)` BEFORE sending each part EXCEPT the first. This simulates typing interval between split messages (per user decision: "多条消息间加延迟模拟打字间隔"). Use a simple loop index check: `if (i > 0) await sleep(1000)`. Apply to both the target branch and the session branch.

In `loop.ts`:

- Record `const loopStartTime = Date.now()` at the start of `run()`, right after the type guard.
- After the loop completes (after the try/catch block, before recording agent summary), if the agent sent messages (`hasSent` is true) OR is about to send fallback text, compute reply delay:
  - `const elapsed = Date.now() - loopStartTime`
  - `const contentLength = sentContent.length` (already computed)
  - `const typingMs = Math.min(contentLength * 50, 3000)` — 50ms per char, capped at 3s
  - `const delay = Math.max(0, typingMs - elapsed)` — subtract inference time to avoid double-latency (per Pitfall 4 in research)
  - If `delay > 0`, `await sleep(delay)` before the fallback send. Import `sleep` from "koishi".
- For the fallback text send path: apply the delay BEFORE `session.send(fallbackText)`.
- For the tool-call send path: the delay is already "natural" since LLM inference takes time, and inter-part delay is handled in send_message. No additional delay needed here — the `hasSent` path just skips to summary recording.
  </action>
