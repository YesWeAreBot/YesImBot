- Keep `Mustache.escape = (text) => text` for no HTML escaping.
- Remove `IRenderer` implementation (just export the class directly).

<task type="auto">
  <name>Task 2: PromptService Rewrite with Named Injection Points</name>
  <files>core/src/services/prompt/service.ts, core/src/services/prompt/index.ts</files>
  <action>
**service.ts** — Rewrite PromptService keeping the Service subclass pattern:

1. **Storage:** Replace `private injections: Injection[]` with `private injections = new Map<InjectionPoint, InjectionEntry[]>()` initialized with empty arrays for all 6 points.

2. **inject(ctx, point, entry)** method:
   - Signature: `inject(ctx: Context, point: InjectionPoint, entry: InjectionEntry): () => void`
   - Check for duplicate name within the point's list — if found, log warning via `this.logger.warn()` and return a no-op disposer.
   - Push entry to the point's list.
   - Create dispose function that splices the entry from the list.
   - Register `ctx.on("dispose", dispose)` on the CALLER's ctx for auto-cleanup.
   - Return the dispose function for manual removal.

3. **registerPartial(name, content)** method:
   - Store in `private partials = new Map<string, string>()`.
   - Last registration wins (overrides). Log info when overriding.

4. **resolveOrder(entries)** private method:
   - Takes `InjectionEntry[]`, returns ordered `InjectionEntry[]`.
   - Implement Kahn's algorithm for topological sort on before/after constraints.
   - On cycle detection: log warning with involved names, fall back to original array order.
   - Typical list size is 1-3 entries, so simplicity over performance.

5. **render(templateName, initialScope?)** method:
   - Returns `Promise<Section[]>` (not string).
   - For each injection point (in INJECTION_POINTS order):
     - Resolve ordering via `resolveOrder()`.
     - Render each entry's `renderFn(scope)` with `Promise.allSettled` wrapped in a per-entry timeout (use `this.config.timeout ?? 5000` ms). Skip timed-out entries with warning.
     - Join non-empty results with `\n\n` into `{point}_content` scope variable.
     - Set `has_{point}` boolean scope variable (true if content non-empty).
   - Collect all registered partials + file-loaded partials into one object.
   - Render the main template via `this.renderer.render(template, scope, allPartials)`.
   - Split rendered output into Section[] by parsing the template's partial references — each `{{>name}}` that maps to an injection point becomes a Section with:
     - `name`: the injection point name
     - `content`: that section's rendered content
     - `cacheable`: true for identity/style/core_memories, false for working_memory/environment/extra

   **Simpler approach for Section[] splitting:** Instead of post-hoc parsing, render each section partial individually:
   - For each injection point, render its partial template with the scope → produces one Section per point.
   - Only include sections with non-empty content.
   - This avoids complex string splitting.

6. **renderToString(templateName, initialScope?)** convenience method:
   - Calls `render()`, joins all section contents with `\n\n`, returns string.

7. **Keep existing APIs that still work:**
   - `registerTemplate(name, content)` — unchanged
   - `registerSnippet(name, fn)` — unchanged
   - `getTemplate(name)` — unchanged
   - `removeInjection(name)` — keep for backward compat, removes from all points

8. **Constructor:** Register default system template via `loadTemplate("system")`. Initialize all 6 injection point lists as empty arrays.

9. **Config:** Add optional `timeout?: number` to `PromptServiceConfig` and its Schema.

**index.ts** — Update exports to include new types: `Section`, `InjectionPoint`, `InjectionEntry`, `INJECTION_POINTS`. Remove `IRenderer` export if present.
