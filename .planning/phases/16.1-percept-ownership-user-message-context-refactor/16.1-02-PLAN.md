**horizon/manager.ts:**
- In `toObservations()`, propagate `entry.stage` to the returned observation object for message entries: add `stage: entry.stage` to the message observation return value

**horizon/service.ts:**
- Delete `toStructured()` method entirely
- Remove `StructuredHorizonView` from imports (already removed from types.ts in Plan 01)
- Update `formatHorizonText()`:
  - Split `view.history` observations into two arrays based on `stage` field:
    - `historyObs`: observations where `stage === "active"` (or stage is undefined, for agent summaries)
    - `triggerObs`: observations where `stage === "new"`
  - Format both arrays using `this.formatObservation()`
  - Pass to Mustache.render: `{ environment, activeMembers, hasHistory: historyObs.length > 0, history: formattedHistory, hasTrigger: triggerObs.length > 0, trigger: formattedTrigger }`
  - Remove the old `triggerType`/`triggerMessage` variables from the render scope (trigger info now comes from New-stage observations)
  - Remove the `const p = view.percept as UserMessagePercept` line and its usage

**horizon-view.mustache — rewrite to use XML section tags with history/trigger split:**
```mustache
<environment>
{{#environment}}
{{environment}}
{{/environment}}
{{^environment}}
No environment info available.
{{/environment}}
</environment>

<members>
{{#activeMembers}}
{{activeMembers}}
{{/activeMembers}}
{{^activeMembers}}
No member info available.
{{/activeMembers}}
</members>

<history>
{{#hasHistory}}
{{#history}}
{{{.}}}
{{/history}}
{{/hasHistory}}
{{^hasHistory}}
No recent messages.
{{/hasHistory}}
</history>

<trigger>
{{#hasTrigger}}
{{#trigger}}
{{{.}}}
{{/trigger}}
{{/hasTrigger}}
{{^hasTrigger}}
No new messages.
{{/hasTrigger}}
</trigger>
```

This follows locked decisions: XML tags for sections, environment→members→history→trigger order, placeholder text for empty sections, trigger = New stage messages, history = Active stage messages.
