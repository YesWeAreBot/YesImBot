## Cross-Phase Integration (15/17 wired)

All 17 requirements have cross-phase integration touchpoints. 15 are fully wired. 2 have partial wiring (non-blocking):

1. **SKILL-04 prompt injection point** — `SkillEffect.promptInjections.point` typed as `InjectionPoint` but hardcoded to `"extra"`. Skills cannot target other injection points. Functional but type is misleading.

2. **trait-bound lifecycle** — Declared in `LifecycleStrategy` type and used by `private-chat` skill, but `resolve()` treats it identically to `per-turn`. No distinct runtime behavior.

## E2E Flows (3/3 complete)

1. **Message → Response:** `horizon/message` event → agent `handleEvent` → willingness gate → aggregation window → `buildPercept` → `buildView` → `trait.analyze` → `skill.resolve` → inject effects → `renderToString` → LLM call → `executeActions` → `recordAgentResponse` → trim → continue/stop
2. **Skill hot-reload:** `reload()` → delete file-sourced skills → `loadAllDirs()` → cache-bust `require()` → fresh `SkillDefinition` → next `resolve()` uses new definitions
3. **Sub-plugin unload:** ctx dispose → `inject()` dispose callback → injection removed from map → next `render()` excludes removed injection
