3. `recordMessage()`: change parameter type from `scope: Scope` to `platform: string; channelId: string`. Inside the method, the spread `...data` will now spread `platform` and `channelId` directly. Use: `const entry = { id: Random.id(), type: TimelineEventType.Message, priority: TimelinePriority.Normal, scope: { platform: data.platform, channelId: data.channelId }, stage: data.stage, timestamp: data.timestamp, data: data.data } as unknown as MessageRecord`.
4. `recordAgentResponse()`: same pattern — change `scope: Scope` to `platform: string; channelId: string` in the parameter. Construct entry with `scope: { platform: data.platform, channelId: data.channelId }` cast.
5. `markAsActive()`: change `scope: Scope` to `key: ChannelKey`. The DB query uses `scope` column, so: `const query: Record<string, unknown> = { scope: { platform: key.platform, channelId: key.channelId }, stage: TimelineStage.New }`.
6. `archiveStale()`: same pattern — `scope: Scope` → `key: ChannelKey`, construct DB query with `scope: { platform: key.platform, channelId: key.channelId }`.

**IMPORTANT DB BRIDGE NOTE:** All DB operations must still write/query the `scope` JSON column because the DB schema hasn't changed yet (Phase 28). The TypeScript types (`BaseTimelineEntry`) now have bare fields, but the actual DB rows have `scope: {platform, channelId}`. Use `as unknown as` casts where needed to bridge this gap. Add a comment at the top of the file: `// NOTE: DB column is still "scope" (JSON) — Phase 28 (CTX-08) will migrate to bare columns`.
