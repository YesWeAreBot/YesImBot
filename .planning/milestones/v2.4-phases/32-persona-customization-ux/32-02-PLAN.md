# Plan 32-02: Persona Injection Wiring & Text Assembly

## Goal

Wire the persona plugin to inject assembled persona text into the `soul` injection point via `PromptService.inject()`, positioned after `__role_soul`. After this plan, enabling the plugin in Koishi Console and filling in persona fields (or selecting a preset) causes the persona text to appear in the LLM system prompt as a supplement to SOUL.md content.

## must_haves

- [ ] `PromptService.inject()` is called with `name: '__persona_supplement'` and `after: '__role_soul'`
- [ ] `buildPersonaText()` merges preset defaults with user-provided field overrides (user fields take priority)
- [ ] When all fields are empty and preset is `none`, no injection is registered (no empty fragment in prompt)
- [ ] Semantic prefix distinguishes persona content from SOUL.md (e.g. "以下是补充人格特质：")
- [ ] Plugin disposes cleanly — injection is removed when plugin is disabled

## Context for executor

### Locked decisions (from CONTEXT.md — non-negotiable)

- Use `PromptService.inject()` directly, NOT `SkillRegistry.register()` (research confirmed skills with no conditions/activate never fire)
- Inject into `soul` point, with `after: '__role_soul'` ordering
- SOUL.md takes priority; persona is complementary
- Semantic prefix to distinguish from SOUL.md content

### Reference: RoleService injection pattern

From `core/src/services/role/service.ts` lines 136-141:

```typescript
this.disposers.push(
  this.prompt.inject(this.ctx, "soul", {
    name: "__role_soul",
    renderFn: (scope) => this.renderSafe("SOUL.md", soulContent, scope),
  }),
);
```

### Reference: PromptService.inject() signature

From `core/src/services/prompt/service.ts` line 86:

```typescript
inject(ctx: Context, point: InjectionPoint, entry: InjectionEntry): () => void
```

`InjectionEntry` from `core/src/services/prompt/types.ts`:

```typescript
interface InjectionEntry {
  name: string;
  renderFn: (scope: Record<string, unknown>) => string | Promise<string>;
  before?: string;
  after?: string;
}
```

### Anti-patterns to avoid

- Do NOT use `SkillRegistry.register()` — skills with no `activate` and no `conditions` evaluate to `false` in `resolve()` and never fire
- Do NOT inject when all fields are empty — guard in `apply()` before calling `inject()`
- Do NOT use Mustache templating in persona text — keep it simple, plain string concatenation
- Do NOT inject before `__role_soul` — SOUL.md must come first per locked decision

## Tasks

<task id="32-02-T1" title="Implement buildPersonaText and preset merging">

In `plugins/persona/src/index.ts`, add the `buildPersonaText()` function:

1. **Preset merge logic:**
   - Import `PRESETS` from `./presets`
   - For each field (`name`, `personality`, `tone`, `extra`): use the user-provided value if non-empty, otherwise fall back to the preset value
   - This gives the "auto-fill from preset, user can override" behavior

2. **Text assembly:**
   - Start with semantic prefix: `"以下是补充人格特质："`
   - Append non-empty fields with labels: `名字：{name}`, `核心性格：{personality}`, `语气风格：{tone}`
   - Append `extra` as-is (no label — it's free-form)
   - Join with `\n`
   - If only the prefix remains (all fields empty after merge), return empty string

3. **Function signature:** `function buildPersonaText(config: Config): string`
   - Pure function, no side effects
   - Returns empty string when nothing to inject

</task>

<task id="32-02-T2" title="Wire PromptService.inject() in apply()">

Complete the `apply()` function in `plugins/persona/src/index.ts`:

1. **Build text and guard:**

   ```typescript
   export function apply(ctx: Context, config: Config) {
     const text = buildPersonaText(config);
     if (!text) return; // nothing to inject
     // ...
   }
   ```

2. **Register injection:**

   ```typescript
   ctx["yesimbot.prompt"].inject(ctx, "soul", {
     name: "__persona_supplement",
     after: "__role_soul",
     renderFn: () => buildPersonaText(config),
   });
   ```

   - `inject()` auto-registers a dispose handler via `ctx.on('dispose', ...)` — no manual cleanup needed
   - The `ctx` parameter passed to `inject()` is the caller's context, so when the persona plugin is disabled, the injection is automatically removed

3. **Verify dispose behavior:** The `inject()` method (line 100 of prompt/service.ts) calls `ctx.on('dispose', dispose)` which binds to the caller's lifecycle. When the persona plugin is unloaded, Koishi disposes its context, which triggers the removal. No additional cleanup code is needed.

**Verify:** `yarn typecheck` passes. The plugin can be enabled/disabled without errors.

</task>
