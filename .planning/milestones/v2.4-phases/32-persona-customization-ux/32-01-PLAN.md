# Plan 32-01: Persona Plugin Scaffold, Schema, Presets & i18n

## Goal

Create the `plugins/persona/` workspace package with a complete Koishi config Schema (preset dropdown + 3-4 form fields + large textarea), built-in preset templates, and i18n locale files. After this plan, the plugin loads in Koishi Console and shows a usable config form — but does not yet inject anything into the prompt.

## must_haves

- [ ] `plugins/persona/` exists as a valid Yarn workspace package with correct `package.json`
- [ ] Config Schema renders in Koishi Console with a preset dropdown and 4 form fields (name, personality, tone, extra textarea)
- [ ] 2-3 built-in preset templates defined in `presets.ts`, selectable via the dropdown
- [ ] `zh-CN.json` and `en-US.json` locale files provide descriptions for all config fields
- [ ] `tsconfig.json` extends the base config and compiles cleanly

## Context for executor

### Locked decisions (from CONTEXT.md — non-negotiable)

- Independent plugin, not built into core
- 3-4 core fields: name, core personality (short text), tone/style (short text), free-form supplement (large textarea)
- Preset dropdown that auto-fills fields; 2-3 curated presets
- Switching presets directly overwrites all fields (no confirmation)

### Claude's discretion

- Specific preset template content (style direction and copy)
- Form field i18n description copy
- Plugin internal Schema structure design

### Reference patterns

**Provider plugin package.json** (`providers/provider-openai/package.json`):

```json
{
  "name": "@yesimbot/koishi-plugin-provider-openai",
  "koishi": { "service": { "required": ["yesimbot.model"] } }
}
```

**Schema i18n pattern** (`core/src/index.ts`):

```typescript
Schema.object({
  field: Schema.string().default(""),
})
  .description({ "zh-CN": "分组名", "en-US": "Group Name" } as never)
  .i18n({ "zh-CN": zhCN._config, "en-US": enUS._config });
```

**Preset dropdown pattern** (from research):

```typescript
Schema.union([
  Schema.const("none").description("无预设"),
  Schema.const("friendly").description("活泼友好"),
]).default("none");
```

**tsconfig.json** (`providers/provider-openai/tsconfig.json`):

```json
{
  "extends": "../../tsconfig.base.json",
  "compilerOptions": { "outDir": "./dist", "rootDir": "./src" },
  "include": ["src", "src/**/*.json"]
}
```

<task id="32-01-T1" title="Create plugin package scaffold">

Create `plugins/persona/` with:

1. **`package.json`**:
   - name: `@yesimbot/koishi-plugin-persona`
   - Same structure as `providers/provider-openai/package.json` (files, type, main, module, types, exports, scripts)

2. **`tsconfig.json`**:
   - Extends `../../tsconfig.base.json`

   - `include: ["src", "src/**/*.json"]`
   - No `references` needed (no workspace dependency on shared-model)

**Verify:** `yarn typecheck` passes for the persona package (after T2 creates the source files).

</task>

<task id="32-01-T2" title="Implement Schema, presets, and i18n locale files">

Create the source files:

1. **`src/presets.ts`** — Export a `PRESETS` record with 2-3 preset templates:
   - `none`: all fields empty (the default — user starts from scratch)
   - `friendly`: 活泼友好风格 (lively, friendly, uses casual language)
   - `professional`: 专业沉稳风格 (calm, professional, concise)
   - Each preset is an object: `{ name: string, personality: string, tone: string, extra: string }`
   - Export the `PresetKey` type (union of preset keys)
   - Export a `PersonaFields` interface for the field shape

2. **`src/locales/zh-CN.json`** — Chinese descriptions:

   ```json
   {
     "_config": {
       "preset": "预设模板（选择后将覆盖所有字段）",
       "name": "名字",
       "personality": "核心性格",
       "tone": "语气风格",
       "extra": "自由补充（支持多行文本，可填写对话示例等）"
     }
   }
   ```

3. **`src/locales/en-US.json`** — English descriptions:

   ```json
   {
     "_config": {
       "preset": "Preset template (selecting will overwrite all fields)",
       "name": "Name",
       "personality": "Core personality",
       "tone": "Tone & style",
       "extra": "Free-form supplement (multi-line, conversation examples, etc.)"
     }
   }
   ```

4. **`src/index.ts`** — Plugin entry with Schema only (injection logic comes in Plan 02):
   - `export const name = 'yesimbot-persona'`
   - `export const inject = ['yesimbot.prompt']`
   - Export `Config` interface with fields: `preset`, `name`, `personality`, `tone`, `extra`
   - Export `Config` Schema using `Schema.object({...})` with:
     - `preset`: `Schema.union([Schema.const('none'), Schema.const('friendly'), Schema.const('professional')]).default('none')`
     - `name`: `Schema.string().default('')`
     - `personality`: `Schema.string().default('')`
     - `tone`: `Schema.string().default('')`
     - `extra`: `Schema.string().role('textarea').default('')`
   - Apply `.description({ 'zh-CN': '人设配置', 'en-US': 'Persona' } as never)`
   - Apply `.i18n({ 'zh-CN': zhCN._config, 'en-US': enUS._config })`
   - `export function apply(ctx: Context, config: Config) {}` — empty body for now (Plan 02 fills this)

**Verify:** `tsc --noEmit` passes. Schema structure matches the Koishi Console expectations.

</task>
