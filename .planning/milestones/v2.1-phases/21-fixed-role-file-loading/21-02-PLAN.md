**types.ts** — Define `RoleServiceConfig` interface with:

Export `RoleServiceConfigSchema` using `Schema.object({ rolePath: Schema.path({ filters: ['directory'], allowCreate: true }).default('data/yesimbot/roles') })`.

**service.ts** — `RoleService extends Service<RoleServiceConfig>`:

Follow CLAUDE.md Service Pattern. Key implementation details:

1. **Declaration merging:** `interface Context { 'yesimbot.role': RoleService }`
2. **Static inject:** `['yesimbot.prompt']` — waits for PromptService
3. **Constructor:** `super(ctx, 'yesimbot.role', false)` — async init via `start()`
4. **Logger:** `ctx.logger('role')` — create once per CLAUDE.md Logger Pattern
5. **Fields:** `prompt: PromptService`, `watcher: FSWatcher | null`, `debounceTimer`, `disposers: Array<() => void>`, `lastValid: Map<string, string>` (for error recovery)

6. **`start()` method:**
   - Call `ensureFiles()` to seed missing files from bundled defaults
   - Call `loadAndInject()` to load, render, and inject all role files
   - Call `startWatching()` to set up fs.watch

7. **`ensureFiles()` method:**
   - For each of `['SOUL.md', 'AGENTS.md', 'TOOLS.md']`:
     - Check if user file exists at `join(config.rolePath, name)`
     - If missing: copy from bundled `core/resources/roles/{name}` using `cpSync`
     - If exists: `logger.debug('Role file %s already exists, skipping seed', name)` (never overwrite per user decision)
   - Bundled dir resolved same way as `builtinResourcesDir` in PromptService: `resolve(__dirname, '../'.repeat(__dirname.includes('dist') ? 1 : 2), 'resources/roles')`

8. **`loadAndInject()` method:**
   - Dispose previous injections: `for (const d of this.disposers) d(); this.disposers = [];`
   - Load SOUL.md → inject at `soul` point with name `__role_soul`, `renderFn` uses MustacheRenderer
   - Load AGENTS.md → inject at `instructions` point with name `__role_agents`
   - Load TOOLS.md → inject at `instructions` point with name `__role_tools`, `after: '__role_agents'`
   - For TOOLS.md: if file doesn't exist, silently skip (ROLE-03)
   - For SOUL.md/AGENTS.md: if file doesn't exist AND bundled copy failed, use minimal inline fallback: `"You are {{bot.name}}."` for soul, `"Respond helpfully."` for instructions
   - `loadFile(name)` helper: `readFileSync(join(config.rolePath, name), 'utf-8')` wrapped in try/catch; returns `null` on ENOENT
   - `renderFn` implementation: wrap `this.prompt.renderer` (wait — renderer is private). Instead, use `Mustache.render(content, scope)` directly (Mustache is already a dependency). On render error (malformed template), warn-log and return `lastValid.get(name) ?? content` (raw content as fallback)

   Actually, looking at MemoryService line 174, it uses `Mustache.render(block.content, scope)` directly. Follow the same pattern — import Mustache directly.

9. **`startWatching()` method:**
   - Same pattern as MemoryService.startWatching() (lines 100-117)
   - `watch(config.rolePath, callback)` with 300ms debounce
   - On change: call `loadAndInject()` then `logger.debug('Role files reloaded')`
   - On dispose: clear timer, close watcher

10. **Undefined variable warning (ROLE-05):**
    - After rendering, Mustache already outputs empty string for undefined vars (default behavior)
    - No extra warning logic needed — Mustache handles this. The user decision says "render as empty string + warn log" but implementing variable detection per-render would be expensive. Use Mustache's default empty-string behavior. The warn log can be added later if users request it.
