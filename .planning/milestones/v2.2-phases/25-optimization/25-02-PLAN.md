**In `core/src/services/agent/loop.ts`:**

Replace the `renderToString` call with `render()` to get `Section[]`, then split into stable vs dynamic content.

1. Change the import to include `Section` type:
   ```typescript
   import type { Section } from "../prompt/types";
   ```

2. Replace the existing `systemPrompt` construction (around line 115):
   ```typescript
   // OLD:
   const systemPrompt = await prompt.renderToString("system", { view, percept });

   // NEW:
   const sections = await prompt.render("system", { view, percept });
   const stableContent = sections
     .filter(s => s.name === "soul" || s.name === "instructions")
     .map(s => s.content)
     .join("\n\n");
   const dynamicContent = sections
     .filter(s => s.name === "memory" || s.name === "extra")
     .map(s => s.content)
     .join("\n\n");
   ```

3. Resolve the provider type for the current model. Extract the provider name from `this.config.model`:
   ```typescript
   const providerType = modelService.getProvider(
     (this.config.model ?? "").split(":")[0]
   )?.providerType;
   ```

4. Pass `systemStable` and `systemDynamic` as new fields on `CallParams`. Since `CallParams = CallSettings & Prompt` and `Prompt.system` accepts `string | SystemModelMessage | SystemModelMessage[]`, we build the system param here:
   ```typescript
   import type { SystemModelMessage } from "ai";

   let systemParam: string | SystemModelMessage[];
   if (providerType === "anthropic") {
     systemParam = [
       {
         role: "system" as const,
         content: stableContent,
         providerOptions: {
           anthropic: { cacheControl: { type: "ephemeral" } },
         },
       },
       {
         role: "system" as const,
         content: dynamicContent,
       },
     ];
   } else {
     // All other providers: plain string, stable first for prefix-cache benefit
     systemParam = stableContent + "\n\n" + dynamicContent;
   }
   ```

5. Use `systemParam` in the `callParams`:
   ```typescript
   const callParams: CallParams = {
     system: systemParam,
     messages,
   };
   ```

6. Also update the `systemPrompt` variable used in the debug log and the wrap-up call. Create a `systemPromptString` for debug logging:
   ```typescript
   const systemPromptString = stableContent + "\n\n" + dynamicContent;
   ```
   Use `systemPromptString` in the debug byte-size log. Use `systemParam` (not `systemPromptString`) in the wrap-up `CallParams` at the max-rounds block too.

7. Add debug logging for stable/dynamic byte sizes at debugLevel >= 3:
   ```typescript
   if ((this.config.debugLevel ?? 0) >= 3) {
     this.logLoop.debug(
       `[${percept.traceId}] system_stable_bytes=${Buffer.byteLength(stableContent, "utf8")} system_dynamic_bytes=${Buffer.byteLength(dynamicContent, "utf8")} provider=${providerType ?? "unknown"}`,
     );
   }
   ```

**IMPORTANT:** Do NOT change the wmLines builder (already modified by Plan 01). Do NOT change the model call, parsing, or tool execution logic. Only the system prompt construction and its passing to callParams changes.
