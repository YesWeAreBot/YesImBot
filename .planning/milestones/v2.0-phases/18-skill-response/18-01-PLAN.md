<task type="auto">
  <name>Task 1: Skill types and condition evaluator</name>
  <files>core/src/services/skill/types.ts, core/src/services/skill/condition.ts</files>
  <action>
Create `core/src/services/skill/types.ts` with:

**ConditionNode** — discriminated union for declarative conditions:
- `MatchNode`: `{ match: { dimension: string; value: string } }` — matches a TraitSignal by dimension+value
- `AndNode`: `{ and: ConditionNode[] }` — all children must match
- `OrNode`: `{ or: ConditionNode[] }` — any child must match
- `NotNode`: `{ not: ConditionNode }` — negation
- `ConditionNode = MatchNode | AndNode | OrNode | NotNode`

**LifecycleStrategy**: `"per-turn" | "sticky" | "trait-bound"`

**StyleEffect**: `{ content: string }` — style text to inject

**ToolFilter**: `{ include?: string[]; exclude?: string[] }` — tool filtering; exclude takes priority

**SkillEffects**: `{ prompt?: string; style?: StyleEffect; tools?: ToolFilter }` — per-skill effects

**SkillDefinition**: The complete skill definition object:
- `name: string`
- `description?: string`
- `conditions?: ConditionNode` — declarative condition (optional if activate is provided)
- `activate?: (signals: TraitSignal[]) => boolean` — code activator (optional)
- `lifecycle: LifecycleStrategy`
- `stickyTimeout?: number` — override for sticky default
- `effects: SkillEffects`
- `source: "file" | "plugin"` — origin tracking

**SkillEffect** (merged result for a turn): The output of resolving all active skills:
- `promptInjections: Array<{ skillName: string; point: InjectionPoint; content: string }>`
- `styleOverride: { content: string; specificity: number } | null`
- `toolFilter: { include: string[]; exclude: string[] }`

Import `TraitSignal` from `../shared/types` and `InjectionPoint` from `../prompt/types`.

Create `core/src/services/skill/condition.ts` with two exported functions:

1. `evaluateCondition(node: ConditionNode, signals: TraitSignal[]): boolean`
   - `match`: signals.some(s => s.dimension === node.match.dimension && s.value === node.match.value)
   - `and`: node.and.every(c => evaluateCondition(c, signals))
   - `or`: node.or.some(c => evaluateCondition(c, signals))
   - `not`: !evaluateCondition(node.not, signals)
   - Default: return false

2. `specificity(node: ConditionNode): number`
   - `match`: 1
   - `and`: sum of children specificities
   - `or`: max of children specificities
   - `not`: specificity of inner node
   - Default: 0

3. `filterByConfidence(signals: TraitSignal[], threshold: number): TraitSignal[]`
   - Returns signals with confidence >= threshold

<task type="auto">
  <name>Task 2: File-based skill loader</name>
  <files>core/src/services/skill/loader.ts, core/src/services/skill/index.ts</files>
  <action>
Create `core/src/services/skill/loader.ts`:

Export `loadSkillsFromDir(dir: string, logger: { warn: (msg: string, ...args: unknown[]) => void }): Promise<SkillDefinition[]>`

Implementation (follow MemoryService.loadBlocks pattern from `core/src/services/memory/service.ts`):
1. `readdir(dir, { withFileTypes: true })` — iterate directory entries
2. For each directory entry, look for `SKILL.md` at `join(dir, entry.name, "SKILL.md")`
3. Read and parse frontmatter using the same regex pattern as MemoryService: `/^---\r?\n([\s\S]*?)\r?\n---\r?\n([\s\S]*)$/`
4. Use `js-yaml` `load()` to parse YAML frontmatter
5. Extract from frontmatter: `name` (fallback to directory name), `description`, `conditions` (as ConditionNode), `lifecycle` (default "per-turn"), `stickyTimeout`, `effects.style`, `effects.tools`
6. Markdown body after frontmatter = `effects.prompt` content
7. Check for `scripts/activate.js` in the skill directory — if exists, `require()` it and use its default export as the `activate` function. Clear `require.cache[resolvedPath]` before requiring (for reload support).
8. Set `source: "file"`
9. Wrap each skill load in try/catch — on error, `logger.warn("Skipping malformed skill %s: %s", entry.name, e)` and continue (per user decision: malformed files skipped with logging)

Create `core/src/services/skill/index.ts`:
- Re-export everything from `./types`
- Re-export `evaluateCondition`, `specificity`, `filterByConfidence` from `./condition`
- Re-export `loadSkillsFromDir` from `./loader`
