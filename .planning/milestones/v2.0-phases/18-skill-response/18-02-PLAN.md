  <action>
Create `core/src/services/skill/service.ts` as a Koishi Service subclass (per CLAUDE.md pattern).

**Declaration merging:**
```ts
declare module "koishi" {
  interface Context {
    "yesimbot.skill": SkillRegistry;
  }
}
```

**Config interface:**
```ts
export interface SkillRegistryConfig {
  skillPaths?: string[];        // User custom skill directories
  confidenceThreshold?: number; // Default 0.3
  stickyDefaultTimeout?: number; // Default 3 rounds
}
```
Export `SkillRegistryConfigSchema` using Koishi Schema with:
- `skillPaths`: `Schema.array(Schema.path({ filters: ["directory"], allowCreate: true }))` with empty default
- `confidenceThreshold`: `Schema.number().default(0.3)`
- `stickyDefaultTimeout`: `Schema.number().default(3)`

**Class SkillRegistry extends Service<SkillRegistryConfig>:**

Constructor:
- `super(ctx, "yesimbot.skill", false)` (async init)
- Create logger once: `this.logger = ctx.logger("skill")`
- Initialize internal maps

Private state:
- `skills: Map<string, SkillDefinition>` — all skills (file + plugin)
- `channelState: Map<string, Map<string, ActiveSkillState>>` — per-channel active skill tracking
  - `ActiveSkillState = { lifecycle: LifecycleStrategy; roundsSinceActive: number; stickyTimeout: number }`

**start():**
1. Load built-in skills from `builtinSkillsDir` (resolve relative to `__dirname` same pattern as PromptService.builtinResourcesDir, pointing to `core/resources/skills/`)
2. Load user custom skills from each path in `config.skillPaths`
3. All loaded skills stored in `this.skills` map by name
4. Log count: `this.logger.info("SkillRegistry started, %d skills loaded", this.skills.size)`

**register(def: SkillDefinition): () => void** — Plugin registration API:
- Store in `this.skills` map by `def.name`
- Return dispose function that deletes from map
- Wire dispose to caller's ctx: `this[Context.current]?.on("dispose", dispose)` for auto-cleanup

**reload(): Promise<void>** — Manual hot-reload:
- Remove all file-sourced skills from map: `for (const [k, v] of this.skills) if (v.source === "file") this.skills.delete(k)`
- Re-run directory loading (built-in + user paths)
- Log: `this.logger.info("Skills reloaded, %d skills", this.skills.size)`

**resolve(signals: TraitSignal[], scope: Scope): SkillEffect** — Core activation + merging:
1. Filter signals by confidence threshold: `filterByConfidence(signals, this.config.confidenceThreshold ?? 0.3)`
2. Derive channel key from scope: `${scope.platform}:${scope.channelId}`
3. Get or create channel state map
4. Determine active skills for this turn:
   - For each skill in `this.skills`:
     - Check activation: if skill has `activate` function, call it with filtered signals; else if skill has `conditions`, call `evaluateCondition(conditions, filteredSignals)`; else skip
     - If activated:
       - For `per-turn`: always include
       - For `sticky`: include, reset `roundsSinceActive` to 0
       - For `trait-bound`: include (signal present = active)
     - If NOT activated:
       - For `sticky`: if in channel state, increment `roundsSinceActive`; if >= timeout, remove from state; else still include (grace period)
       - For `per-turn` / `trait-bound`: exclude
5. Build merged SkillEffect from active skills:
   - **Prompt**: For each active skill with `effects.prompt`, add to `promptInjections` array with `point: "extra"` and content wrapped in `<skill name="...">content</skill>` tags
   - **Style**: Collect all active skills with `effects.style`. Compute specificity for each (from their conditions via `specificity()` function). Pick highest specificity; on tie, later registration wins. Set as `styleOverride` with content and specificity.
   - **Tools**: Merge all `effects.tools` — concatenate all `include` arrays, concatenate all `exclude` arrays. Exclude takes priority (applied at consumption time).
6. Return the merged `SkillEffect`

**builtinSkillsDir** constant: Resolve to `core/resources/skills/` using same `__dirname` pattern as PromptService's `builtinResourcesDir`.

Also create the empty `core/resources/skills/` directory (mkdir -p) so the loader doesn't error on start.
