  <name>Task 1: Create trimmer.ts with progressive trimming function</name>
  <files>core/src/services/agent/trimmer.ts</files>
  <action>
Create `core/src/services/agent/trimmer.ts` with:

1. Export `TrimConfig` interface:
   - `charBudget: number` (default used in loop: 30000)
   - `keepLastRounds: number` (default: 2)
   - `softTrimHead: number` (default: 800)
   - `softTrimTail: number` (default: 800)

2. Export `LoopMessage` interface:
   - `role: "user" | "assistant"`
   - `content: string`
   - `_trimState?: "none" | "soft" | "hard"` — tracks degradation to prevent double-trimming

3. Export `trimMessages(messages: LoopMessage[], config: TrimConfig): void` — mutates in-place:
   - Calculate total character count of all messages
   - If under `charBudget`, return immediately
   - Identify rounds: index 0 is always protected (initial user context). Rounds are (assistant, user) pairs starting from index 1: round 1 = [1,2], round 2 = [3,4], etc.
   - Calculate total round count. Protect the last `keepLastRounds` rounds.
   - For eligible rounds (oldest first), apply trimming with priority: trim user messages (tool results) first, then assistant messages.
   - First pass: softTrim all eligible messages with `_trimState === undefined` or `"none"`. Recheck budget after each message. Stop if under budget.
   - Second pass (if still over): hardClear eligible messages with `_trimState !== "hard"`. For user messages, extract tool names from "Tool results:\n[...]" JSON format and replace with `[tool results cleared: name1, name2]`. For assistant messages, replace with `[assistant response cleared]`. Recheck budget after each.

4. Internal helper `softTrim(text: string, head: number, tail: number): string`:
   - If `text.length <= head + tail + 20`, return text unchanged
   - Return `text.slice(0, head) + "\n...(trimmed)...\n" + text.slice(-tail)`

5. Internal helper `hardClearToolResult(content: string): string`:
   - Try to parse tool names from "Tool results:\n[{...}]" format
   - Return `[tool results cleared: name1, name2]` or `[tool results cleared]` on parse failure
  </action>
