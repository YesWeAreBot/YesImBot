**Create 6 section partial files in `core/resources/templates/partials/`:**

Each partial conditionally renders only when its injection point has content. Use `{{#has_X}}...{{/has_X}}` guard and `{{{X_content}}}` (triple-brace, no escaping) for the content.

- `identity.mustache`: Wraps in `<identity>` tags. Default content (from current system.mustache identity block) goes into the PromptService constructor as a default injection — NOT in the partial template. The partial just renders `{{{identity_content}}}`.
  ```mustache
  {{#has_identity}}
  <identity>
  {{{identity_content}}}
  </identity>
  {{/has_identity}}
  ```

- `style.mustache`: Same pattern with `<style>` tags and `{{{style_content}}}`.
- `core-memories.mustache` (filename uses hyphen, partial name uses underscore `core_memories`): `<core_memory>` tags with `{{{core_memories_content}}}`. Note: the loader must map hyphenated filenames to underscored partial names, OR register partials with underscored names in the service constructor.
- `working-memory.mustache`: `<working_memory>` tags with `{{{working_memory_content}}}`.
- `environment.mustache`: `<environment>` tags with `{{{environment_content}}}`.
- `extra.mustache`: `{{{extra_content}}}` only (no wrapper tags — extra is freeform).

**Important:** Register these partials in PromptService constructor using `loadPartial()` and `registerPartial()` with underscore names matching the injection points. E.g., `this.registerPartial("identity", loadPartial("identity"))`.

```typescript
// Old:
this.ctx["yesimbot.prompt"].inject("core-memory", 10, (scope) => { ... });

this.ctx["yesimbot.prompt"].inject(this.ctx, "core_memories", {
  name: "core-memory",
```

**Wait — clarification on data flow:** Per RESEARCH.md Pattern 5, HorizonView data is per-request and goes into render scope (not persistent injections). The environment/working_memory partials render from scope variables, not from inject(). So:

1. ThinkActLoop passes `structured.environment`, `structured.members`, `structured.history` as scope to `renderToString()`.
2. PromptService's render() makes these available as scope variables.
3. The environment partial template renders `{{{environment_content}}}` — but this content comes from the scope, not from injections.

**Resolution:** The environment partial needs to render the structured data directly from scope. Update the environment partial to render structured environment/members/history data:

```mustache
{{#has_environment}}
<environment>
{{#environment}}
Location: {{environment.name}} ({{environment.type}}{{#environment.platform}}, {{environment.platform}}{{/environment.platform}})
{{/environment}}
{{#members.length}}
Active members: {{#members}}{{name}}{{#badge}} [{{badge}}]{{/badge}}{{^last}}, {{/last}}{{/members}}
{{/members.length}}
{{#history.length}}
--- Message History ---
{{#history}}
[{{time}}] {{#isSummary}}[Bot Summary]: {{content}}{{/isSummary}}{{^isSummary}}{{#isBot}}[Bot] {{/isBot}}{{sender}}: {{content}}{{/isSummary}}
{{/history}}
{{/history.length}}
</environment>
{{/has_environment}}
```

Actually, this is getting complex for Mustache. **Simpler approach:** In ThinkActLoop, format the structured data into text strings and pass as scope variables that the partials render directly:

```typescript
const structured = horizon.toStructured(view);
// Format environment text for the partial
const envLines: string[] = [];
if (structured.environment) {
  const e = structured.environment;
  envLines.push(`Location: ${e.name} (${e.type}${e.platform ? `, ${e.platform}` : ""})`);
}
if (structured.members.length) {
  envLines.push(`Active members: ${structured.members.map(m => m.badge ? `${m.name} [${m.badge}]` : m.name).join(", ")}`);
}
if (structured.history.length) {
  envLines.push("--- Message History ---");
  for (const h of structured.history) {
    const prefix = h.isSummary ? "[Bot Summary]" : h.isBot ? `[Bot] ${h.sender}` : h.sender;
    envLines.push(`[${h.time}] ${prefix}: ${h.content}`);
  }
}

const systemPrompt = await prompt.renderToString("system", {
  view,
  environment_content: envLines.join("\n"),
  has_environment: envLines.length > 0,
});
```

This keeps partials simple (just `{{{environment_content}}}` in tags) and formatting logic in TypeScript where it's easier to maintain.

The user message for the LLM call changes too — instead of `formatHorizonText(view)` producing the full context blob, the context is now in the system prompt's environment section. The user message should just be the trigger:

```typescript
const messages = [{
  role: "user" as const,
  content: `[${userPercept.triggerType}] ${userPercept.payload.content}`,
}];
```

**PromptService constructor — Register default identity and style injections:**

Move the current default identity/style text from system.mustache into default injections registered in the PromptService constructor (using the service's own ctx). These serve as fallback content when no plugin overrides them:

```typescript
// In constructor, after initializing injection points:
this.inject(this.ctx, "identity", {
  name: "__default_identity",
  renderFn: () => `你是 {{view.self.name}}，一个有血有肉的人，正在和朋友们聊天。\n{{#view.environment}}\n你现在在「{{view.environment.name}}」里。\n{{/view.environment}}\n你不是 AI 助手，不要表现得像客服或机器人。`,
});
```

Wait — injection renderFn returns plain text, not Mustache templates. The scope variables like `view.self.name` won't be resolved in injection content. **Better approach:** Register these as snippets that read from scope, or make the default identity injection read from scope:

```typescript
this.inject(this.ctx, "identity", {
  name: "__default_identity",
  renderFn: (scope) => {
    const view = scope.view as { self?: { name?: string }; environment?: { name?: string } };
    const name = view?.self?.name ?? "";
    const env = view?.environment?.name;
    let text = `你是 ${name}，一个有血有肉的人，正在和朋友们聊天。`;
    if (env) text += `\n你现在在「${env}」里。`;
    text += `\n你不是 AI 助手，不要表现得像客服或机器人。`;
    return text;
  },
});

this.inject(this.ctx, "style", {
  name: "__default_style",
  renderFn: () => [
    "- 说话自然随意，像真人发消息一样，不用敬语",
    "- 回复简短，不要长篇大论，除非对方明确需要详细解释",
    "- 可以有自己的情绪和看法，不必总是附和",
    "- 不要在回复里加多余的标点或格式，保持口语化",
  ].join("\n"),
});
```

Also register a default `how_you_work` style injection or fold it into identity. Per user decisions, there are only 6 injection points and `how_you_work` isn't one. Fold the how_you_work content into the default identity injection as a second paragraph.
