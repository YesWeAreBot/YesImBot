# Phase 16.1 Plan 01: Percept Ownership Refactor Summary

**Horizon broadcasts raw HorizonMessageEvent, agent owns Percept construction with configurable aggregation window and direct-message bypass**

## Performance

- **Duration:** 6 min
- **Started:** 2026-02-21T12:34:05Z
- **Completed:** 2026-02-21T12:40:00Z
- **Tasks:** 2
- **Files modified:** 6

## Accomplishments
- Horizon listener is now a pure data layer: records timeline + broadcasts raw `horizon/message` events
- Agent module owns all decision logic: willingness gating, aggregation window, Percept construction
- Type system enforces the new boundary via `HorizonEventMap` declaration merging
- Aggregation window is configurable via `AgentCoreConfig.aggregationWindow` (default 1500ms)
- Direct messages bypass aggregation for immediate Percept construction

## Task Commits

1. **Task 1: Refactor horizon types and listener to broadcast raw events** - `20ddfdb` (refactor)

## Files Created/Modified
- `core/src/services/horizon/types.ts` - Added HorizonMessageEvent, HorizonEventMap, BasePerceptRef; removed Percept types and StructuredHorizonView
- `core/src/services/horizon/listener.ts` - Removed buildPercept/schedulePercept/pendingPercepts; emits horizon/message
- `core/src/services/horizon/service.ts` - Updated buildView signature, removed toStructured(), fixed imports
- `core/src/services/agent/service.ts` - Added Percept types, handleEvent, buildPercept, pendingWindows, aggregationWindow config
- `core/src/services/agent/loop.ts` - Import Percept from agent module, use formatHorizonText instead of toStructured
- `core/src/services/plugin/types.ts` - Import Percept from agent module

## Decisions Made
- Percept types placed directly in agent/service.ts for locality (not a separate types file)
- HorizonView.percept narrowed to BasePerceptRef (minimal: id, type, scope, timestamp) to prevent circular imports
- Removed toStructured() from HorizonService since StructuredHorizonView was deleted; loop.ts now uses formatHorizonText() directly
- buildView() accepts BasePerceptRef + optional runtime parameter instead of UserMessagePercept
