  <action>
Create `plugins/core/src/services/memory/` directory with three files:

**types.ts:**
- `MemoryBlock` interface: `label: string`, `title?: string`, `description?: string`, `content: string`, `filename: string`
- `MemoryConfig` interface: `coreMemoryPath?: string`, `memoryCharLimit?: number`

**service.ts:**
- `MemoryService extends Service<MemoryConfig>` with `static inject = ["yesimbot.prompt"]`
- Declaration merging: `Context["yesimbot.memory"]: MemoryService`
- Constructor: `super(ctx, "yesimbot.memory", false)` (async init via start())
- `start()`: call `loadBlocks()`, call `startWatching()`, call `registerInjection()`
- `loadBlocks()`: use `readdir` on `config.coreMemoryPath`. Filter `.md`/`.txt` files only. Sort alphabetically. For each file, `readFile` and call `parseFrontmatter()`. If no files found OR no coreMemoryPath configured, use a single default persona block (inline constant, NOT a file). Store blocks in `private blocks: MemoryBlock[]`.
- `parseFrontmatter(raw: string)`: regex split on `---` delimiters, parse YAML section with `js-yaml`'s `load()`. Return `{ meta, content }`. If no frontmatter match, return raw content with empty meta. Derive `label` from `meta.label` or filename (strip extension).
- `startWatching()`: if `config.coreMemoryPath` is set, use `node:fs` `watch()` on the directory. Debounce 300ms with a timer. On change, call `loadBlocks()`. Register `ctx.on("dispose", ...)` to close watcher and clear timer.
- `registerInjection()`: call `this.ctx["yesimbot.prompt"].inject("core-memory", 10, (scope) => {...})`. Inside the renderFn: get blocks, if none return "". Apply `memoryCharLimit` (default 4000) — include whole blocks until limit reached, skip remaining (log warning for skipped). For each included block, render template variables in content via `Mustache.render(block.content, scope)`. Format as XML: `<core_memory>` wrapper, each block as `<{label}>` with optional `<title>` child, then content, then `</{label}>`. Return the XML string.
- Default persona constant (inline string, neutral/generic style per user decision):
```
## 关于我

我是一个友好的聊天伙伴。我会根据对话内容自然地回应，保持真诚和适度的好奇心。

## 交流风格

- 自然对话，不过度正式也不过度随意
- 根据对方的语气和话题调整回应方式
- 有自己的想法，但尊重不同观点
```

**index.ts:**
- Re-export: `export * from "./types"` and `export { MemoryService, type MemoryConfig } from "./service"`

Key rules:
- Use `import { load as yamlLoad } from "js-yaml"` (transitive dep, no install needed)
- Use `import Mustache from "mustache"` (existing dep)
- Use `import { readdir, readFile } from "node:fs/promises"` and `import { watch } from "node:fs"`
- Logger: `const log = ctx.logger("yesimbot.memory")` stored once, used throughout
- Do NOT use gray-matter — hand-roll frontmatter parsing with regex + js-yaml

In `plugins/core/src/services/prompt/service.ts`, update `DEFAULT_SYSTEM_TEMPLATE` to include the injections placeholder AFTER the existing `</how_you_work>` closing tag.

Add this block at the end of the template string (before the closing backtick):
```
{{#injections}}

{{{injections}}}
{{/injections}}
```

This uses Mustache section syntax: the block only renders when `injections` is truthy (i.e., when at least one injection produced content). Triple-mustache `{{{injections}}}` prevents double-escaping of XML tags in memory blocks. Note: Mustache.escape is already set to identity, but triple-mustache is more explicit about intent.

The existing render() method already builds `scope["injections"]` from sorted injection fragments. No other changes needed to service.ts.
