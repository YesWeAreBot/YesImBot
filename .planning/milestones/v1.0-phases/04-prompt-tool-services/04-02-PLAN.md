1. Add `"experimentalDecorators": true` to `tsconfig.base.json` compilerOptions — required for legacy TS decorators used by @Tool/@Action.

2. Create `plugins/core/src/services/plugin/types.ts`:
   - `FunctionType` enum: `Tool = "tool"`, `Action = "action"`
   - `ToolResult<T = unknown>` interface: `{ status: "success" | "failed"; result?: T; error?: string }`
   - `FunctionDefinition` interface: `{ name: string; description: string; type: FunctionType; parameters: Schema; handler: (params: Record<string, unknown>, ctx: FunctionContext) => Promise<ToolResult> }`
   - `FunctionContext` interface: `{ session?: Session; view?: HorizonView; percept?: Percept; [key: string]: unknown }` — import Session from koishi, HorizonView/Percept from horizon types
   - `PluginMetadata` interface: `{ name: string; description: string; builtin?: boolean }`
   - `PluginServiceConfig` interface: `{ defaultTimeout?: number }` — default 30000ms

3. Create `plugins/core/src/services/plugin/utils.ts`:
   - `Success<T>(result?: T): ToolResult<T>` — returns `{ status: "success", result }`
   - `Failed(message: string): ToolResult` — returns `{ status: "failed", error: message }`

4. Create `plugins/core/src/services/plugin/schema.ts`:
   - `schemaToJSONSchema(schema: Schema): Record<string, unknown>` — recursive converter from Koishi Schema to JSON Schema object
   - Handle types: `object` (iterate `schema.list` for properties, check `meta.required`), `string`, `number`, `boolean`, `array` (recurse inner), `const`/`union` (enum)
   - Koishi Schema stores type in `schema.type`, children in `schema.list`, inner type in `schema.inner`, description in `schema.meta.description`, required in `schema.meta.required`
   - Return a plain object (no external JSON Schema type dependency needed)

5. Create `plugins/core/src/services/plugin/decorators.ts`:
   - `@Metadata(meta: PluginMetadata)` class decorator — sets `target.prototype.__pluginMetadata = meta`
   - `@Tool(opts: { name, description, parameters })` method decorator — pushes `{ ...opts, type: FunctionType.Tool, methodKey }` to `target.__staticTools ??= []`
   - `@Action(opts: { name, description, parameters })` method decorator — pushes `{ ...opts, type: FunctionType.Action, methodKey }` to `target.__staticActions ??= []`
   - `defineTool(name, description, parameters, handler): FunctionDefinition` — functional registration helper
   - `defineAction(name, description, parameters, handler): FunctionDefinition` — functional registration helper
   - `withInnerThoughts(params: Record<string, Schema>): Schema` — wraps params in `Schema.object({ inner_thoughts: Schema.string().description("Deep inner monologue private to you only."), ...params })`
   - All decorators use legacy TS decorator signatures (not TC39 stage 3)

6. Create `plugins/core/src/services/plugin/base-plugin.ts`:
   - `abstract class Plugin`: - `metadata: PluginMetadata` — read from `this.constructor.prototype.__pluginMetadata` in constructor - `tools: Map<string, FunctionDefinition>` — populated from `__staticTools` in constructor by binding method handlers - `actions: Map<string, FunctionDefinition>` — populated from `__staticActions` in constructor - `getFunctions(): Map<string, FunctionDefinition>` — returns merged Map of tools + actions - Constructor reads decorator metadata arrays from prototype, creates FunctionDefinition entries with `handler` bound to `this[methodKey]` - Supports functional registration: `registerTool(def: FunctionDefinition)` and `registerAction(def: FunctionDefinition)` for defineTool/defineAction results
     </action>

   - Internal state: `plugins: Map<string, Plugin>` — plugin instances keyed by metadata.name
   - Public API:
     - `register(plugin: Plugin)` — adds to plugins Map, logs registration
     - `unregister(name: string)` — removes from plugins Map
     - `async invoke(name: string, params: Record<string, unknown>, context?: FunctionContext): Promise<ToolResult>` — finds function by name across all plugins, calls handler with timeout (use `Promise.race` with `setTimeout` for `this.config.defaultTimeout ?? 30000`), wraps errors in `Failed()`. Return the ToolResult.
     - `getDefinition(name: string): FunctionDefinition | undefined` — finds function definition by name across all plugins
     - `getTools(context?: FunctionContext): Array<{ type: "function"; function: { name: string; description: string; parameters: Record<string, unknown> } }>` — traverses all plugins, for each FunctionDefinition calls `schemaToJSONSchema(def.parameters)`, returns ai-sdk compatible tool format array
     - `listPlugins(): string[]` — returns plugin names

   - Handler: extracts `content` from params, calls `ctx.session.send(content)` via FunctionContext.session, returns `Success()` on success, `Failed(error.message)` on error
