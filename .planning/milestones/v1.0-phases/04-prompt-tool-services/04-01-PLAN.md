1. Create `plugins/core/src/services/prompt/service.ts`:
   - `PromptService extends Service` (Koishi), registered as `"yesimbot.prompt"` with `immediate: true`
   - Declaration merging: `declare module "koishi" { interface Context { "yesimbot.prompt": PromptService } }`
   - Internal state:
     - `templates: Map<string, string>` — named template strings (e.g. "system", "persona", "rules")
     - `snippets: Map<string, Snippet>` — named data providers
     - `injections: Injection[]` — plugin-contributed fragments
     - `renderer: MustacheRenderer` — instantiated in constructor
   - Public API methods:
     - `registerTemplate(name: string, content: string)` — stores template; config override takes priority (check `this.config.templates?.[name]` first)
     - `registerSnippet(name: string, fn: Snippet)` — stores snippet by name
     - `inject(name: string, priority: number, renderFn: Snippet)` — adds an Injection to the list
     - `removeInjection(name: string)` — removes injection by name
     - `async render(templateName: string, initialScope?: Record<string, unknown>): Promise<string>` — main render method:
       a. Resolve template content: `this.config.templates?.[templateName] ?? this.templates.get(templateName)` (config > built-in priority per locked decision)
       b. Call `buildScope(initialScope)` to evaluate snippets — only evaluate snippets whose keys appear in the template (optimization from research: `getRequiredVariables()` scans for `{{key}}` patterns)
       c. Evaluate injections sorted by priority (ascending), each rendered via its `renderFn(scope)`, filter empty results, join with `\n\n`, and set as `scope.injections` (or a named key like `scope.rules_injections`)
       d. Call `this.renderer.render(templateContent, scope, partials)` where partials = `Object.fromEntries(this.templates)`
       e. Return rendered string
   - Private helpers:
     - `getRequiredVariables(template: string): Set<string>` — regex scan for `{{variable}}` and `{{{variable}}}` patterns, extract variable names
     - `async buildScope(initialScope, requiredVars): Record<string, unknown>` — merge initialScope, then for each snippet whose key is in requiredVars, call snippet and set result via `setNestedProperty(scope, key, result)` (supports dot-notation keys like `bot.name`)
     - `setNestedProperty(obj, path, value)` — splits path by `.` and sets nested value
