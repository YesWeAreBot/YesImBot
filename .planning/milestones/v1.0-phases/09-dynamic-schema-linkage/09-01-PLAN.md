<task type="auto">
  <name>Task 2: Implement refreshSchemas() in ModelService</name>
  <files>plugins/core/src/services/model/service.ts</files>
  <action>
In `plugins/core/src/services/model/service.ts`:

1. Add a private `refreshSchemas()` method that:
   - Iterates all registered providers via `this.providers`
   - For each provider, calls `provider.listModels()` to get `Record<string, ModelInfo>`
   - Builds Schema options: `Schema.const('providerName:modelId').description(info.description ?? modelId)` for each model
   - Appends a custom free-text input: `Schema.string().description('Custom model (provider:model)')` at the end of the union
   - Calls `this.ctx.schema.set('registry.chatModels', Schema.union([...options, customInput]).default(''))` per v3-dev reference pattern
   - If no models registered, still set the schema with just the custom input option

2. Modify `registerProvider()`:
   - After `this.providers.set(name, provider)`, call `this.refreshSchemas()`
   - Register a dispose hook: `this.ctx.on('dispose', () => { ... })` that calls `this.unregisterProvider(name)` when the caller's context is disposed. Use the caller context pattern — the `ctx` available in `registerProvider` is the caller's fork, so `ctx.on('dispose', ...)` auto-cleans on provider plugin unload.

   IMPORTANT: The dispose hook must use the service's own context (`this.ctx`) for the schema refresh, but the dispose listener should be on the caller's context. Since `registerProvider` is called by provider plugins through the service proxy, `this[Context.current]` gives the caller's context. Use:
   ```ts
   const caller = this[Context.current];
   caller.on('dispose', () => {
     this.unregisterProvider(name);
   });
   ```
   Import `Context` if not already imported (it is — from koishi).

3. Modify `unregisterProvider()`:
   - After `this.providers.delete(name)`, call `this.refreshSchemas()`

4. Call `this.refreshSchemas()` in the constructor after queue initialization — sets initial empty schema so `Schema.dynamic('registry.chatModels')` resolves immediately.

5. Import `Schema` from `koishi` (add to existing import).

Note: `ModelService` already has `static inject = ["console"]`. The `console` service provides `ctx.schema` in Koishi. This is required for `ctx.schema.set()` to work.
  </action>
