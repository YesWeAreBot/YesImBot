Create listener.ts porting from dev version (`YesImBot-dev/packages/core/src/services/horizon/listener.ts`) with v4 additions (trigger classification, message aggregation).

**EventListener class:**

- Constructor takes `ctx: Context`, `events: EventManager`, `config: { allowedChannels: Array<{ platform: string, type: string, id: string }>, keywords?: string[], aggregationWindow?: number }`.
- Create logger once: `this.logger = ctx.logger('horizon')`.
- Track pending percepts for aggregation: `private pendingPercepts = new Map<string, { timer: NodeJS.Timeout, percept: UserMessagePercept }>()`.

**Public methods:**

1. `start()` — registers Koishi middleware and after-send hook. Store disposers for cleanup.

2. `stop()` — calls all disposers, clears all pending aggregation timers.

**Private methods:**

3. `isChannelAllowed(session: Session): boolean` — port from dev service.ts. Checks session against `config.allowedChannels` array. Match by platform + type (private vs guild) + id (wildcard '\*' or exact channelId/guildId/userId).

4. `classifyTrigger(session: Session): TriggerType` — v4 addition per research:
   - If `session.isDirect` return `'direct'`.
   - If `session.quote?.user?.id === session.bot.selfId` return `'reply'`.
   - If session.elements has `at` element with `id === session.selfId` return `'mention'`.
   - If `config.keywords?.some(kw => session.content?.includes(kw))` return `'keyword'`.
   - Return `'random'`.

5. `recordUserMessage(session: Session): Promise<void>` — port from dev:
   - Call `this.events.recordMessage(...)` with scope from session, stage New, timestamp from session, data with messageId/senderId/senderName/content.
   - Also call `this.updateMemberInfo(session)` if session.guildId exists.

6. `recordBotSentMessage(session: Session): Promise<void>` — port from dev:
   - Guard: return if no session.content or no session.messageId.
   - Call `this.events.recordMessage(...)` with stage Active, senderId = session.bot.selfId, senderName from bot user.

7. `updateMemberInfo(session: Session): Promise<void>` — port from dev:
   - Upsert EntityRecord with id = `${platform}:${author.id}@guild:${guildId}`, type 'member', name from nick/name, attributes with roles and platform.
   - Use try/catch, log errors.

8. `buildPercept(session: Session): UserMessagePercept` — builds Percept object:
   - Generate id via `Random.id()`.
   - Set type, scope, priority 5, timestamp, triggerType from classifyTrigger.
   - Set payload with messageId, content, sender, channel.
   - Set runtime.session.

9. `schedulePercept(channelKey: string, percept: UserMessagePercept): void` — message aggregation:
   - If `session.isDirect` (check via percept.scope.isDirect), emit immediately via `ctx.emit('horizon/percept', percept)`, return.
   - For group chats: check existing pending. If exists, clear old timer, preserve higher-priority trigger (priority order: mention > reply > keyword > random — use a numeric map).
   - Use `ctx.setTimeout()` (not raw setTimeout) per research pitfall 4 — Koishi auto-cancels on dispose.
   - Window = `config.aggregationWindow ?? 1500` ms.
   - On timer fire: delete from map, emit percept.

**Middleware registration (in start()):**

```
ctx.middleware(async (session, next) => {
  if (!this.isChannelAllowed(session)) return next()
  if (session.author?.isBot) return next()
  await this.recordUserMessage(session)
  await next()  // let other middleware run first
  const percept = this.buildPercept(session)
  const channelKey = `${session.platform}:${session.channelId}`
  this.schedulePercept(channelKey, percept)
})
```

**After-send registration (in start()):**

```
ctx.on('after-send', (session) => {
  if (!this.isChannelAllowed(session)) return
  this.recordBotSentMessage(session)
}, true)
```

Store both disposers in an array for cleanup in stop().
