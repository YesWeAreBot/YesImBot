  <name>Task 1: Create HorizonService with DB schema and HorizonView building</name>
  <files>plugins/core/src/services/horizon/service.ts</files>
  <action>
Create service.ts as a Koishi Service subclass, porting from dev version with v4 simplifications.

**Koishi declaration merging** (at top of file):

- `Context['yesimbot.horizon']`: HorizonService
- `Tables['yesimbot.timeline']`: TimelineEntry
- `Tables['yesimbot.entity']`: EntityRecord
- `Events['horizon/percept']`: (percept: Percept) => void

**HorizonConfig interface:**

- `allowedChannels: Array<{ platform: string, type: string, id: string }>`
- `keywords?: string[]`
- `aggregationWindow?: number`
- `historyLimit?: number` (default 30)

**HorizonService class extends Service<HorizonConfig>:**

- `static inject = ['database']`
- Constructor: `super(ctx, 'yesimbot.horizon', false)` — false because async start() needed for DB schema.
- Create `this.events = new EventManager(ctx)` in constructor.
- Create logger once: `this.logger = ctx.logger('horizon')`.

**start() method:**

1. Register DB models via `ctx.model.extend`:
   - `'yesimbot.timeline'`: id string(32), scope object, type string(32), priority unsigned, stage string(16), timestamp timestamp, data json. Primary: id, autoInc: false.
   - `'yesimbot.entity'`: id string(64), type string(32), name string(255), parentId string(255), refId string(255), attributes json, updatedAt timestamp. Primary: id.
2. Create and start EventListener: `new EventListener(ctx, this.events, this.config).start()`.
3. Log startup.

**stop() method:** Log shutdown.

**Public methods:**

1. `buildView(percept: UserMessagePercept): Promise<HorizonView>` — per research Pattern 6 + user decision:
   - Query Timeline via `this.events.query({ scope: { platform, channelId }, types: [Message, AgentSummary], limit: config.historyLimit ?? 30, orderBy: 'asc' })`.
   - Convert to observations via `this.events.toObservations(entries)`.
   - Get environment via `this.getEnvironment(percept.scope)`.
   - Get entities via `this.getEntities(percept.scope)`.
   - Return HorizonView object with percept, self info (from session bot), environment, entities, history.

2. `getEnvironment(scope: Scope): Promise<Environment | null>` — query EntityRecord with type='channel' and matching id. Convert to Environment. Return null if not found (stub for v4, will be enriched later).

3. `getEntities(scope: Scope): Promise<Entity[]>` — query EntityRecords with parentId matching guild scope. Map to Entity interface. Return empty array if no guild.

4. `formatObservation(obs: Observation, selfId?: string): string` — v4 simple chat-log format:
   - For message: `[HH:MM] {prefix}{senderName}: {content}` where prefix is `[Bot]` if sender.id matches selfId.
   - For agent.summary: `[HH:MM] [Bot Summary]: {summary}`.

5. `formatHorizonText(view: HorizonView): string` — formats full HorizonView as text for LLM:
   - Section 1: Environment info (name, description if available).
   - Section 2: Active entities (names list).
   - Section 3: Message history (formatted observations).
   - Section 4: Trigger context (trigger type, triggering message).

Do NOT include ChatModeManager, CommandService, MemoryService, AssetService — all deferred per user decisions.
