  <action>
Add runtime dependencies to `plugins/core/package.json`:
- `"ai": "^4.1.0"` (for generateText, streamText)
- `"p-queue": "^8.0.0"` (for request queue)

Create `plugins/core/src/services/model-service.ts`:
- `ModelService` class extends Koishi `Service` AND implements `IModelService` from shared-model
- Constructor: `constructor(ctx: Context, config: ModelServiceConfig)` calling `super(ctx, 'model-service', true)` (immediate=true since no async init needed)
- Add declaration merging: `declare module 'koishi' { interface Context { 'model-service': ModelService } }`
- `ModelServiceConfig` interface: `{ defaultProvider?: string, defaultModel?: string, fallbackChains?: Record<string, Array<{ provider: string, model: string }>>, concurrency?: number }`
- Provider registry: `Map<string, IModelProvider>` with register/unregister/get/list methods
- **Wrapped call** `call(providerName: string | undefined, modelId: string | undefined, params: object)`:
  - Resolve provider/model from args or defaults
  - Queue via p-queue (default concurrency=5)
  - Call `generateText({ model, ...mergedParams })` where mergedParams = provider defaults merged with caller overrides
  - On error: classify with `classifyError`, if TRANSIENT or RATE_LIMIT attempt fallback chain
  - Track usage (store cumulative token counts in a simple Map)
  - Return ai-sdk result
- **Wrapped stream** `streamCall(providerName, modelId, params)`: Same pattern but uses `streamText` from ai-sdk. No queue wrapping for streams (they are long-lived). Fallback on initial connection error only.
- **Meta call** `getModel(providerName, modelId)`: Returns `{ model: LanguageModelV3, defaultParams: ModelDefaultParams }` for direct ai-sdk usage by caller
- **Query methods**: `getModelInfo(providerName, modelId)`, `getUsage()` returns cumulative usage map
- Fallback: `handleFallback(primaryKey, params, error)` — looks up chain from config, tries each in order, throws original error if all fail
- Log provider register/unregister via `ctx.logger('model-service')`

Update `plugins/core/src/index.ts`:
- Import ModelService
- Add to Config schema: `defaultProvider: Schema.string()`, `defaultModel: Schema.string()`, `fallbackChains: Schema.dict(Schema.array(...))`, `concurrency: Schema.number().default(5)`
- In `apply()`: load ModelService as sub-plugin via `ctx.plugin(ModelService, config)` — Service 子类本身就是合法插件，构造时自动注册到所有上下文
- 无需手动 `ctx.provide()` 或赋值 `ctx['model-service']`，Service 基类在构造函数中自动处理
- 卸载时 Service 基类自动移除服务，依赖方插件自动回滚，无需手动 dispose
